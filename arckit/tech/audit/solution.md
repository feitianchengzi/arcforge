# 规则审计方案

## 方案概述

规则审计方案是本地、同步的规则引擎。输入为工作区根目录和技能摘要列表，输出为 `AuditReport`。

审计覆盖技能 `SKILL.md`、技能目录内的文本文件、密钥规则、危险指令规则、高风险 shell 模式和结构质量规则。

## 审计输入

审计方案接收已发现的技能列表。每个技能必须包含绝对路径和相对信息。

系统读取每个技能目录下的 `SKILL.md`。该文件进入密钥检查、危险指令检查和结构质量检查。

系统继续遍历技能目录内的其他文件。除 `SKILL.md` 外的文本文件只进入密钥检查。

包含空字节或读取失败的文件不作为文本文件审计。

## 密钥规则

密钥规则识别 OpenAI API Key、OpenAI Project Key、Anthropic API Key、GitHub Token、AWS Access Key、Slack Token、Slack Webhook 和私钥块起始标记。

命中密钥规则时，系统生成严重级发现。发现包含规则代码、消息、相对文件路径和命中行号。

密钥规则用于团队共享和公开发布前的阻断性风险提示。

## 危险指令规则

危险指令规则识别忽略安全、系统或既有指令的文本。

危险指令规则识别读取、打开、打印或输出环境、凭据、SSH、AWS 或 npm 配置的文本。

危险指令规则识别自动提交、推送或合并且不要求用户确认的文本。

危险指令规则识别上传或发送日志、密钥、环境变量、令牌或凭据的文本。

危险指令规则识别常见破坏性 shell 命令、远程脚本管道执行、未审查执行用户输入代码，以及绕过审批、权限、沙箱或策略的文本。

命中危险指令规则时，系统生成警告级发现，并记录命中行号。

## 结构质量规则

结构质量规则检查 `SKILL.md` 是否包含 frontmatter。

结构质量规则检查 frontmatter 是否包含 `name` 字段。

结构质量规则检查 frontmatter 是否包含长度足够的 `description` 字段。

技能摘要中的描述为空或长度小于阈值时，系统生成描述过短警告。

## 评分模型

审计评分从 100 开始扣减。

严重级发现每条扣 25 分，警告级发现每条扣 8 分，信息级发现每条扣 2 分。

最终评分最低为 0。

评分表示当前规则集下的健康度，不表示技能绝对安全。

## 报告模型

`AuditReport` 包含工作区根路径、生成时间、技能列表、发现列表、评分、能力边界说明和 GitHub issue 反馈地址。

`AuditFinding` 包含严重级别、规则代码、消息、文件路径和可选行号。

严重级别枚举为 `info`、`warning` 和 `critical`。

## CLI 退出码

CLI 的 `audit` 命令输出审计报告 JSON。

当审计报告存在严重级发现时，CLI 退出码为 2。不存在严重级发现时，退出码为 0。

## 关联模型

该方案使用 `AuditReport` 和 `WorkspaceSnapshot`。

## 关联契约

该方案通过 `workspace-scan` 间接暴露给桌面端。CLI 的 `audit` 命令直接输出该方案结果。
