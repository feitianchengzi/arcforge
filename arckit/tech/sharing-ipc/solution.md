# 共享与 IPC 方案

## 方案概述

共享与 IPC 方案覆盖 Electron 安全桥接、系统环境检测、GitHub 来源下载、发布计划生成、共享执行和漂移差异窗口。

渲染层只通过 preload 暴露的方法调用主进程。主进程负责文件系统、Git、目录选择器和窗口创建。

## IPC 边界

preload 暴露 `window.skillops` 对象。该对象包含选择工作区、扫描工作区、初始化配置、保存配置、查询默认目标、查询环境、下载来源、生成发布计划、共享项目、应用配置组、漂移检查和打开差异窗口等方法。

主进程通过 `ipcMain.handle` 注册对应 channel。每个 handler 返回 Promise 结果或抛出错误。

渲染层不访问 Node.js API。BrowserWindow 启用上下文隔离并关闭 Node 集成。

## 默认目标

默认目标由主进程根据当前用户主目录生成。

Codex 目标路径为用户主目录下 `.codex/skills`。

Claude 目标路径为用户主目录下 `.claude/skills`。

Cursor 目标路径为用户主目录下 `.cursor/skills`。

## 环境检测

环境检测执行 `git --version`。

Git 可用时，结果包含平台、架构、Git 可用状态和 Git 版本。

Git 不可用时，结果包含平台、架构、Git 不可用状态和错误信息。

模型为 `EnvironmentStatus`。

## 来源解析

来源解析接受 SSH Git URL、完整 HTTP Git URL、`github.com/owner/repo`、`owner/repo` 和 GitHub tree/blob URL。

GitHub tree/blob URL 解析为克隆地址、引用和子目录。

非 GitHub HTTP URL 按普通 Git URL 处理，不解析子目录。

来源为空或格式不符合规则时，系统抛出远端地址错误。

## 来源下载

来源下载把解析后的仓库克隆到应用数据目录下的 `sources` 缓存。

缓存目录名称由仓库名称和来源摘要组成。

缓存目录已存在时，系统执行快进拉取。

来源包含引用时，系统检出该引用。

来源包含子目录时，返回该子目录作为项目根路径。子目录不存在时，系统抛出错误。

## 发布计划

发布计划由 core 的 `createPublishPlan` 生成。

计划输入为工作区根目录和可见性。主进程先扫描工作区，再以配置和技能列表生成 `PublishPlan`。

计划包含文件清单、安装命令和检查项，不执行远端写入。

## 共享执行

共享执行先解析远端来源，再扫描当前工作区。

系统在应用数据目录下的 `share-worktrees` 准备共享仓库工作树。工作树目录由仓库名和仓库地址摘要组成。

工作树不存在时，系统克隆远端仓库。工作树存在时，系统拉取远端引用信息。

目标分支优先使用来源引用。没有来源引用时，系统读取远端默认分支；无法读取时使用 `main`。

远端分支存在时，系统基于远端分支重建本地分支并快进拉取。远端分支不存在时，系统创建本地分支。

目标子目录由来源子目录、共享目标模式和项目名计算。直接模式使用来源子目录；按项目名模式追加项目名。

系统清理相对路径中的空段、当前目录段和上级目录段。

## 共享文件写入

系统保存本地共享设置，包括远端仓库、共享目标模式和项目名。

系统写入共享仓库使用的配置。共享仓库配置中的 `teamRepo` 指向可安装的远端项目引用。

系统替换目标路径下的技能来源目录，复制当前工作区技能来源目录，写入 `skillops.config.json`。

当前工作区存在 README 时，系统复制 README；不存在时生成基础 README。

系统在 README 中插入或替换 SkillOps 共享区块。该区块包含可见性、桌面端使用流程、配置组和 CLI 安装命令。

## Git 执行

系统对目标路径执行 `git add`。

Git 状态存在变更时，系统使用用户提交信息创建提交。提交信息为空时使用默认提交信息。

Git 状态没有变更时，系统不创建提交，并记录没有文件变更。

系统推送当前分支到远端并设置 upstream。

共享结果模型为 `ShareResult`。

## 漂移差异窗口

主进程根据 `DriftReport` 渲染只读 HTML 差异窗口。

差异窗口展示配置组、目标目录、总数、变更数、一致数和文件级哈希摘要。

窗口禁用 Node 集成并启用上下文隔离。

## 关联契约

该方案由 `source-download`、`system-environment`、`publish-plan`、`publish-share`、`profile-apply`、`profile-drift` 和差异窗口调用共同支撑。
