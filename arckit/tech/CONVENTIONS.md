# 技术文档约定

## 文档边界

`arckit/tech/` 记录当前已采用的技术方案、数据模型和 IPC 契约。方案文档描述系统事实、模块职责、数据流和错误语义。

## 技术状态

技术方案使用以下状态：

- ✅ 已采用
- 🔬 调研中
- 📋 调研完成
- ❌ 已废弃
- ⚪ 待定

数据模型和契约使用以下状态：

- ✅ 已定义
- 🟡 设计中
- ⚪ 待定义

## 模型与契约

跨领域 DTO、配置结构和标准结果模型统一维护在 `_shared/models/`。渲染层、主进程、CLI 和 core 之间的调用边界统一维护在 `_shared/contracts/`。

契约表示 Electron IPC/preload 或 CLI/core 调用边界，不暗示存在 HTTP 后端。

## 引用规则

方案文档通过模型名和契约名引用 `_shared` 资产。依赖关系集中维护在 `_map/RELATIONS.md`。

## 内容约束

方案文档不包含完整函数实现、脚本或临时实现建议。YAML 模型和契约按 Schema 完整描述字段、枚举、必填项和错误语义。

## 命名规则

技术领域目录使用短横线命名。模型文件使用 PascalCase。契约文件使用小写短横线并以业务动作命名。
