# 桌面应用

## 功能概述

桌面应用提供 SkillOps 的主要交互界面。用户可以添加本地或 GitHub Skill 项目，查看项目健康度，管理配置组，应用技能到目标目录，检查漂移并准备共享发布。

## 应用外壳

桌面端使用独立窗口运行。渲染进程没有直接 Node.js 访问能力，所有文件系统、Git 和工作区操作通过预加载桥接接口发送到主进程。

窗口最小尺寸保证项目列表、顶部状态区和功能页签可以稳定显示。

应用启动时读取本地保存的语言设置、最近项目、当前项目、目标历史和项目界面状态。

## 项目列表

侧边栏展示最近使用的 Skill 项目。每个项目显示名称、技能数量和审计评分。

最近项目记录保存互斥的来源元数据。本地项目记录 `localSourcePath`，GitHub 项目记录 `githubSourceUrl`，同一记录不同时保存两种来源。

用户可以从侧边栏移除项目记录。移除当前项目时，系统清空当前项目状态并回到未选择状态。

GitHub 来源下载过程中，侧边栏显示临时项目记录。下载失败时，临时项目保留失败状态和错误信息。

项目切换先更新当前项目上下文，再执行扫描。旧项目的异步扫描结果或扫描错误不会覆盖用户随后切换到的新项目状态。

## 添加项目

用户可以通过本地目录选择器添加已有 Skill 项目。

用户可以输入 GitHub 来源下载共享 Skill 项目。下载完成后，系统打开下载得到的项目根目录。

添加项目弹窗以本地来源和 GitHub 来源二选一的方式工作。用户选择本地时只展示目录选择入口；选择 GitHub 时只展示远端地址输入和下载入口。

未选择项目时，主内容区展示添加项目空状态和期望目录结构。

## 顶部状态

顶部区域显示当前项目名称、操作状态、Git 环境状态和常用操作。

项目详情页内容区顶部保留类似标题栏高度的窗口拖拽区域。该区域支持拖拽窗口和双击窗口系统行为，不覆盖项目摘要、阶段入口或具体操作按钮。

系统检测 Git 是否可用。Git 可用时显示版本信息；Git 不可用时提示下载和共享 GitHub 项目不可用。

用户可以重新扫描当前项目。重新扫描会更新快照、审计结果和页面展示。

用户可以初始化配置。初始化会写入默认 SkillOps 配置并重新扫描。

当前项目存在快照时，顶部操作区提供编辑项目来源入口。用户可以把当前项目重新指向一个本地文件夹，或重新输入一个 GitHub 来源并下载；保存后的最近项目记录只保留当前选择的来源类型和原始来源地址。

## 项目总览

项目总览展示技能数量、共享资产数量、审计评分、严重问题数量和警告数量。

总览提供通往审计、配置组、应用目标和共享页面的操作入口。

总览展示当前项目最近的目标应用历史。

## 技能页面

技能页面以内置文件查看和编辑器展示当前工作区 `sourceDir` 目录。

文件查看器使用父子文件夹树展示技能、共享资产和技能内部文件。目录支持展开和收起。

文件树支持按当前配置组过滤。配置组包含全部技能时，文件树展示完整 `sourceDir`；配置组只包含部分技能时，文件树展示对应技能目录，并保留共享资产目录。

用户选择文件后，右侧编辑区读取并展示文件内容。文本文件支持编辑、保存和重载。二进制文件和超过编辑限制的大文件不进入编辑器。

技能页面支持把当前文件树和选中文件打开到独立编辑窗口。独立窗口沿用内置页面的配置组过滤、选中文件、目录收起状态、滚动位置和语言文案。

技能页的文件目录和编辑内容区域各自滚动。阶段入口和项目摘要不随文件内容滚动。

## 审计页面

审计页面展示审计评分、严重问题数量、警告数量和发现列表。

审计页面高亮展示当前审计机制和能力边界说明，并提供可点击的 GitHub issue 入口，用于反馈问题或请求更强审计覆盖。

每条发现展示严重级别、规则代码、说明、文件路径和可选行号。

没有发现时，页面显示无发现状态。

## 配置组页面

配置组页面展示并编辑当前工作区配置组。

用户可以新增配置组、删除配置组、修改名称和描述、选择全部技能或具体技能。

保存配置组后，系统写入配置文件、重新扫描项目并保持用户选择的配置组。

## 应用目标页面

应用目标页面允许用户选择配置组和目标位置。

用户可以选择 Agent 目标和自定义目标。Agent 目标支持用户级部署，也支持与项目目录组合后部署到项目内对应 agent 技能目录。

选择项目目录时，系统要求至少选择一个 Agent。未选择项目目录时，选中的 Agent 使用用户级目标目录。

自定义目标按用户选择的目录原样部署。只要存在至少一个 Agent 目标或一个自定义目标，目标组合即可保存、应用和检查漂移。

系统目录选择器允许选择隐藏文件夹，以支持 `.codex`、`.claude`、`.cursor` 等 agent 技能目录。

页面展示即将应用的技能数量、来源项目和目标路径。

用户可以执行应用操作。应用完成后，页面展示复制和跳过数量，并更新漂移报告。

用户可以单独检查漂移。漂移报告展示每个技能或资产的状态和差异统计。

用户可以打开完整漂移差异窗口，查看文件级缺失、变更和额外文件。

## 共享页面

共享页面支持生成发布计划和执行共享。

用户可以选择私有团队仓库或公开发布计划。生成计划后，页面展示安装命令和检查项。

用户可以输入远端仓库、共享目标模式、项目文件夹名和提交信息。

当前项目位于本地 Git 仓库且仓库存在远端时，共享目标编辑弹窗展示“共享到同一个仓库”开关。开关开启后远端仓库输入禁用，输入框不默认填写当前仓库地址；当前 Git 远端地址只作为占位提示或源信息展示。系统默认使用本地 Git 远端地址，并把目标路径固定为当前 Skill 项目在仓库内的相对路径。

本地 Git 仓库没有远端时，页面不展示同仓库开关。本地 Git 仓库存在多个远端时，页面在开关之外展示远端源列表。用户手动输入与本地远端相同的仓库地址时，页面自动开启同仓库模式。

共享页面提供共享目标漂移检查。普通共享目标检查当前配置组技能和共享资产相对目标仓库路径的文件差异；同仓库共享目标检查当前 Skill 项目路径下的本地 Git 变更。检查结果展示变更数量和目标路径，并可打开完整漂移差异窗口。

页面检测 GitHub 登录状态和目标仓库权限。检测完成后，页面展示推荐交付方式，包括目标仓库 Pull Request、fork Pull Request、直接推送共享分支和本地分支。

用户在执行共享前查看执行确认。确认内容包括目标仓库、目标分支、共享分支、交付方式、配置组、变更摘要和 Pull Request 标题。

共享完成后，页面展示远端地址、分支、目标路径、提交状态、推送状态、Pull Request 链接和执行消息。

权限不足、GitHub 未登录、推送失败或 Pull Request 创建失败时，页面保留表单输入、本地分支和提交结果，并提供 fork PR、本地分支、复制命令或重试入口。

## 设置

设置对话框支持切换 English 和简体中文。

设置对话框提供反馈入口。用户点击后打开 GitHub issue 页面。

语言设置持久化在本地浏览器存储中。切换语言后，界面文案立即更新。

## 成功标准

用户能够在一个桌面工作区中完成扫描、审计、配置组管理、目标应用、漂移检查和共享发布。

桌面端能够在没有直接 Node.js 权限的渲染进程中完成本地文件和 Git 工作流。

项目级界面状态能够在重新打开项目时恢复。
