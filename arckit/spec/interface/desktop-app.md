# 桌面应用

## 功能概述

桌面应用提供 SkillOps 的主要交互界面。用户可以添加本地或 GitHub Skill 项目，查看项目健康度，管理配置组，应用技能到目标目录，检查漂移并准备共享发布。

## 应用外壳

桌面端使用独立窗口运行。渲染进程没有直接 Node.js 访问能力，所有文件系统、Git 和工作区操作通过预加载桥接接口发送到主进程。

窗口最小尺寸保证项目列表、顶部状态区和功能页签可以稳定显示。

应用启动时读取本地保存的语言设置、最近项目、当前项目、目标历史和项目界面状态。

## 项目列表

侧边栏展示最近使用的 Skill 项目。每个项目显示名称、技能数量和审计评分。

用户可以从侧边栏移除项目记录。移除当前项目时，系统清空当前项目状态并回到未选择状态。

GitHub 来源下载过程中，侧边栏显示临时项目记录。下载失败时，临时项目保留失败状态和错误信息。

## 添加项目

用户可以通过本地目录选择器添加已有 Skill 项目。

用户可以输入 GitHub 来源下载共享 Skill 项目。下载完成后，系统打开下载得到的项目根目录。

未选择项目时，主内容区展示添加项目空状态和期望目录结构。

## 顶部状态

顶部区域显示当前项目名称、操作状态、Git 环境状态和常用操作。

系统检测 Git 是否可用。Git 可用时显示版本信息；Git 不可用时提示下载和共享 GitHub 项目不可用。

用户可以重新扫描当前项目。重新扫描会更新快照、审计结果和页面展示。

用户可以初始化配置。初始化会写入默认 SkillOps 配置并重新扫描。

## 项目总览

项目总览展示技能数量、共享资产数量、审计评分、严重问题数量和警告数量。

总览提供通往审计、配置组、应用目标和共享页面的操作入口。

总览展示当前项目最近的目标应用历史。

## 技能页面

技能页面展示扫描得到的技能列表。

每个技能展示名称、描述、相对路径、目标标识、版本、是否包含引用资料和是否包含脚本。

技能页面展示共享资产列表。共享资产说明其会随配置组同步到目标目录。

当没有共享资产时，页面显示空状态。

## 审计页面

审计页面展示审计评分、严重问题数量、警告数量和发现列表。

每条发现展示严重级别、规则代码、说明、文件路径和可选行号。

没有发现时，页面显示无发现状态。

## 配置组页面

配置组页面展示并编辑当前工作区配置组。

用户可以新增配置组、删除配置组、修改名称和描述、选择全部技能或具体技能。

保存配置组后，系统写入配置文件、重新扫描项目并保持用户选择的配置组。

## 应用目标页面

应用目标页面允许用户选择配置组和目标位置。

用户可以选择内置 agent 目标、项目目标或自定义目标。

页面展示即将应用的技能数量、来源项目和目标路径。

用户可以执行应用操作。应用完成后，页面展示复制和跳过数量，并更新漂移报告。

用户可以单独检查漂移。漂移报告展示每个技能或资产的状态和差异统计。

用户可以打开完整漂移差异窗口，查看文件级缺失、变更和额外文件。

## 共享页面

共享页面支持生成发布计划和执行共享。

用户可以选择私有团队仓库或公开发布计划。生成计划后，页面展示安装命令和检查项。

用户可以输入远端仓库、共享目标模式、项目文件夹名和提交信息。

共享完成后，页面展示远端地址、分支、目标路径、提交状态、推送状态和执行消息。

## 设置

设置对话框支持切换 English 和简体中文。

语言设置持久化在本地浏览器存储中。切换语言后，界面文案立即更新。

## 成功标准

用户能够在一个桌面工作区中完成扫描、审计、配置组管理、目标应用、漂移检查和共享发布。

桌面端能够在没有直接 Node.js 权限的渲染进程中完成本地文件和 Git 工作流。

项目级界面状态能够在重新打开项目时恢复。
