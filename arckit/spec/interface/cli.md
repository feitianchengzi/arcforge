# 命令行接口

## 功能概述

命令行接口为 ArcForge 的核心工作流提供结构化执行层。用户可以在终端、脚本或 CI 中扫描当前目录、执行审计、检查当前 Git checkout 更新、把当前项目技能归并到另一个 Skill 项目、维护应用关系、检查漂移、应用配置组、生成发布计划和执行 Git 共享交付。

Agent Skill 默认通过 CLI 获取无副作用查询结果，并在用户确认后通过 CLI 执行写入、归并、应用、更新和共享动作。CLI 的结构化输出是 coding agent 编排 ArcForge 工作流的基础。

CLI 不提供初始化命令，也不提供来源登记命令。当前项目状态由配置保存、归并和应用关系保存动作自动创建或更新。

## 通用行为

CLI 接收命令名和参数。没有命令或请求帮助时，系统输出使用说明。

所有针对当前项目的命令默认以当前目录作为工作区根目录。用户可以通过 `--root` 指定其他目录。

扫描、审计和归并命令支持 `--source-dir`。该参数覆盖当前工作区配置中的技能来源目录，但不改变工作区根目录。用户在项目根目录下维护 `.codex/skills` 等项目内 agent skill 时，可以保持 `--root` 为项目根目录，并通过 `--source-dir .codex/skills` 指定技能来源。

成功命令把结果序列化为格式化 JSON 输出到标准输出。未知命令或执行错误输出错误消息，并设置失败退出状态。

需要写入文件、更新 Git checkout、重新应用应用关系或修改远端仓库的命令要求显式确认。

CLI 输出面向机器消费保持稳定字段，面向人工终端保持可读错误信息。Agent Skill 使用 CLI JSON 结果生成当前项目状态、风险摘要、漂移摘要和下一步建议。

## 扫描与审计

`scan` 命令读取当前目录配置、发现技能、发现共享资产并执行审计，输出完整工作区快照。

`audit` 命令执行扫描，并只输出审计报告。审计报告包含能力边界说明和 GitHub issue 反馈地址。

当用户通过 `--source-dir` 指定项目内 agent skill 目录时，扫描和审计结果的工作区根目录仍是项目根目录，技能相对路径以该来源目录开头。

当审计报告中存在严重级发现时，`audit` 使用严重问题退出状态；没有严重级发现时使用成功退出状态。

## Git 来源检查

`source status` 命令检查当前 `--root` Git checkout 相对上游的 ahead、behind、dirty 和 fast-forward 可更新状态。

`source update` 命令在显式确认后对当前 `--root` 执行 fast-forward-only 更新。

该命令不依赖来源登记或应用关系。只要当前目录是有 remote 和上游分支的 Git checkout，就可以检查更新。

当本地存在未提交变更、本地领先上游、处于 detached HEAD 或无法解析上游分支时，更新命令返回失败结果并停止。

## 归并命令

`merge plan` 命令把当前项目中的技能计划归并到另一个 Skill 项目目录。计划动作不写文件。

`merge run` 命令在显式确认后执行无冲突归并。

用户必须通过 `--to` 指定目标 Skill 项目。`--to` 可以是本地目录、GitHub shorthand、GitHub URL、Git URL 或 SSH Git URL。

用户必须通过 `--target-path` 指定目标 Skill 项目内的写入目录。用户可以通过 `--skills` 指定技能列表，通过 `--profile` 指定目标项目配置组，通过 `--target` 指定当前项目记录的应用目标目录。

`--target-path` 表示正式 Skill 项目中的父级写入目录。系统会在该目录下追加 skill 名称作为最终目录。用户归并 `project-showcase-video` 到标准技能目录时应传入 `--target-path skills`，而不是 `--target-path skills/project-showcase-video`。

归并计划包含新增、一致和冲突技能。存在冲突时，执行动作停止，不覆盖目标项目中的不同内容。

归并执行成功后，系统更新目标 Skill 项目配置组，并在当前项目中创建或合并应用关系。

归并命令不自动提交、推送或创建 Pull Request。

## 应用关系命令

`applied list` 命令列出当前项目保存的应用关系。

`applied add` 命令手动保存当前项目对某个来源 Skill 项目的应用关系。用户必须通过 `--from` 指定来源 Skill 项目。

`applied remove` 命令删除应用关系。删除只更新用户级状态，不删除目标目录中的已应用文件。

`applied drift` 命令基于应用关系检查漂移。漂移检查以来源 Skill 项目为来源，以记录的目标目录为目标。

`applied run` 命令在显式确认后基于应用关系重新应用来源 Skill 项目配置组。

## 一次性应用与漂移

`apply` 命令从指定 Skill 项目或当前目录应用配置组到目标目录。

`drift` 命令比较来源配置组和目标目录。

用户通过 `--from` 指定来源 Skill 项目。未指定 `--from` 时，命令以当前 `--root` 作为来源。

`apply` 要求 `--target`。`apply --save` 会把本次来源、配置组、目标目录和技能选择保存为当前项目应用关系。

`drift` 要求 `--target`，并输出文件级漂移报告。

Agent Skill 在项目内 skill 优化场景中使用一次性应用和漂移命令比较正式 Skill 项目与目标项目 agent 目录。存在保存价值的关系通过应用关系命令记录。

Agent Skill 串联归并、应用和漂移时按顺序执行写入步骤。归并完成前不执行应用，应用完成前不执行漂移。

## 发布计划与共享

`publish-plan` 命令执行扫描，并基于当前目录输出发布计划 JSON。

发布计划包含文件清单、安装命令和检查项。发布计划只做发布就绪说明，不登记来源，也不执行远程写入。

`share plan` 和 `share run` 命令执行扫描、生成发布计划、检测 GitHub 权限，并按用户指定或系统推荐的交付方式执行共享。

共享命令支持目标仓库、可见性、配置组、目标路径模式、项目文件夹名、提交信息、共享分支名、交付方式、确认参数、同仓库共享开关和同仓库远端名称。

共享执行远端写入时，会创建或更新共享分支、提交变更、推送分支，并在交付方式要求时创建 Pull Request。

## 环境诊断命令

`doctor` 命令检查本地运行环境、Git、CLI shim 和可选集成工具，输出环境状态 JSON。

## Agent Skill 编排

CLI 支持 Agent Skill 把用户意图路由到具体命令。扫描、审计、发布计划、来源状态和漂移检查作为默认无副作用动作执行。

应用配置组、归并执行、来源更新、共享执行和远端写入动作保留显式确认。Agent Skill 在调用这些动作前说明目标目录、来源项目、配置组、远端仓库或 Git 分支。

CLI 的错误结果包含足够的上下文，使 Agent Skill 能够说明失败原因和可恢复动作。常见可恢复动作包括切换目标目录、处理归并冲突、清理 dirty Git checkout、登录 GitHub、改用本地分支或打开桌面端查看完整差异。

## 弃用说明

CLI 不提供 `init` 命令。当前项目状态由具体动作自动创建。

CLI 不提供 `sources` 命令。来源不再作为独立登记实体存在。

CLI 不提供 `promote` 命令。项目内技能正式化通过 `merge` 完成。

CLI 不提供 `apply-profile` 命令。配置组应用通过 `apply` 命令处理。

## 成功标准

用户能够通过 CLI 在本地或 CI 中获得与桌面端一致的扫描和审计结果。

用户能够通过 Agent Skill 间接使用 CLI 完成主要 ArcForge 工作流，不需要记忆具体命令名称和参数。

用户能够直接使用本地或远程 Skill 项目完成归并、应用和漂移检查，不需要先登记来源。

用户能够把当前项目技能归并到另一个 Skill 项目，并让当前项目自动记录为应用目标。

用户能够基于应用关系检查漂移和重新应用来源 Skill 项目中的正式版本。

用户能够独立检查和更新当前 Git checkout。

用户能够用 JSON 结果驱动后续发布、漂移检查、应用或共享交付脚本。
