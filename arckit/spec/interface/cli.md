# 命令行接口

## 功能概述

命令行接口为 SkillOps 的核心工作流提供 JSON 输出。用户可以在终端、脚本或 CI 中初始化配置、扫描工作区、执行审计、生成发布计划、检查漂移和应用配置组。

## 通用行为

CLI 接收命令名和参数。没有命令或请求帮助时，系统输出使用说明。

所有核心命令默认以当前目录作为工作区根目录。用户可以通过根目录参数指定其他目录。

成功命令把结果序列化为格式化 JSON 输出到标准输出。

未知命令返回错误并设置失败退出状态。

命令执行过程中抛出的错误会输出错误消息，并设置失败退出状态。

## 初始化命令

初始化命令在目标工作区写入默认配置。

命令输出初始化后的配置 JSON。

默认配置版本为 1，来源目录为 `skills`，默认配置组选择全部技能并包含 Codex、Claude 和 Cursor。

## 扫描命令

扫描命令读取配置、发现技能、发现共享资产并执行审计。

命令输出完整工作区快照 JSON。

快照包含根路径、配置、技能列表、共享资产列表和审计报告。

## 审计命令

审计命令执行扫描，并只输出审计报告 JSON。

审计报告 JSON 包含能力边界说明和 GitHub issue 反馈地址，明确当前审计是本地规则扫描，不是完整安全审计，可能漏报或误报。

当审计报告中存在严重级发现时，命令使用严重问题退出状态。

当不存在严重级发现时，命令使用成功退出状态，即使存在警告级发现。

## 发布计划命令

发布计划命令执行扫描，并基于扫描结果输出发布计划 JSON。

用户可以通过可见性参数指定私有或公开发布。缺省可见性为私有。

发布计划包含文件清单、安装命令和检查项。

发布计划命令可以检查 GitHub 登录状态和目标仓库权限。权限检查结果包含推荐交付方式、可用交付方式和不可用原因。

## 共享执行命令

共享执行命令执行扫描、生成发布计划、检测 GitHub 权限，并按用户指定或系统推荐的交付方式执行共享。

命令支持目标仓库、可见性、配置组、目标路径模式、项目文件夹名、提交信息、共享分支名、交付方式、确认参数、同仓库共享开关和同仓库远端名称。

交付方式包括目标仓库 Pull Request、fork Pull Request、直接推送共享分支和本地分支。

命令默认不执行远端写入，除非用户提供确认参数。未提供确认参数时，命令输出执行预览 JSON，并使用成功退出状态。

命令执行远端写入时，会创建或更新共享分支、提交变更、推送分支，并在交付方式要求时创建 Pull Request。

使用同仓库共享开关时，目标仓库参数不是必填。命令使用当前工作区所在 Git 仓库的所选远端，固定提交当前 Skill 项目相对路径，并把当前分支推送到该远端。

命令输出共享结果 JSON，包含权限状态、交付方式、目标远端、共享分支、提交哈希、推送状态、Pull Request URL、手动恢复命令和错误阶段。

GitHub 未登录、token scope 不足、无仓库权限或 Pull Request 创建失败时，命令返回失败退出状态，并保留本地分支和可恢复命令。

## 来源维护命令

来源维护命令用于 GitHub 或 Git 来源的 Skill 项目。

`source status` 命令以当前目录或根目录参数指定的工作区为输入，读取本地 Git 仓库和当前分支，获取上游 refs，并输出来源更新状态 JSON。

来源更新状态包含 Git 根目录、工作区相对路径、当前分支、上游分支、远端名称、远端地址、本地 HEAD、上游 HEAD、ahead commit 数、behind commit 数、上次 fetch 时间、距离上次 fetch 的时长、未提交变更状态、是否可快进更新和检查时间。

`source update` 命令必须接收确认参数。确认后，系统重新执行状态检查，并仅在本地落后上游、没有本地领先提交且没有未提交变更时执行 fast-forward 更新。

当当前目录不在 Git 仓库中、当前仓库处于 detached HEAD、当前分支没有上游、存在未提交变更或存在本地领先提交时，来源维护命令返回错误。

来源维护命令不替代 Git 客户端。它只提供 SkillOps 需要的检查点和安全的快进更新路径，让用户在看到落后 commit 数之后自主决定是否更新。

## 漂移命令

漂移命令执行扫描，并对指定配置组和目标目录生成漂移报告。

配置组缺省为 `default`。目标目录缺省为 `.skillops/skills`。

命令输出漂移报告 JSON。

配置组不存在时，命令返回错误。

## 应用配置组命令

应用配置组命令执行扫描，并把指定配置组应用到目标目录。

配置组缺省为 `default`。目标目录缺省为 `.skillops/skills`。

命令输出应用结果 JSON，包含复制和跳过的技能与共享资产。

配置组不存在时，命令返回错误。

## 自动化约束

CLI 输出结构稳定，适合 CI 或脚本消费。

CLI 不打开桌面窗口，不使用界面本地状态。

CLI 共享发布支持预览优先。默认命令只输出计划和权限检查；带确认参数的执行命令可以完成 Git 共享推送和 Pull Request 创建。

CLI 来源维护支持检查优先。状态命令只报告上游差异；更新命令必须显式确认，并且只允许 fast-forward 更新。

CLI 在 CI 中运行时不进入交互式确认。缺少确认参数、GitHub 凭据或必要权限时，命令输出结构化错误并使用失败退出状态。

## 成功标准

用户能够通过 CLI 在本地或 CI 中获得与桌面端一致的扫描和审计结果。

用户能够用退出状态阻断包含严重审计问题的自动化流程。

用户能够用 JSON 结果驱动后续发布、漂移检查或安装脚本。

用户能够在终端中预检 GitHub 权限、选择共享交付方式，并在确认后创建 Pull Request 或保留本地共享分支。

用户能够在终端中检查 GitHub 来源项目相对上游落后多少 commit，并在确认后执行安全的快进更新。
