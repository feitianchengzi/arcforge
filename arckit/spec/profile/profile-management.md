# 配置组管理

## 功能概述

配置组管理功能让用户为同一个 Skill 项目定义多组技能集合。每个配置组可用于不同 agent、项目、团队或发布场景。

## 配置组模型

配置组包含名称、可选描述、技能选择列表和目标 agent 标识列表。

名称是配置组的主要识别字段。系统通过名称查找配置组，并在配置组不存在时返回错误。

技能选择列表支持通配选择全部技能。列表为空时，该配置组不选择任何技能。

目标 agent 标识用于表达该配置组面向的应用对象。当前默认目标包括 Codex、Claude 和 Cursor。

## 默认配置组

当工作区没有配置文件时，系统提供默认配置组。默认配置组名称为 `default`，选择全部技能，并包含 Codex、Claude 和 Cursor 目标。

初始化工作区时，系统写入同等语义的默认配置组。该配置组描述为当前工作区的默认技能集合。

## 技能选择

系统根据配置组的技能选择列表从工作区技能列表中选取技能。

当技能选择列表包含通配项时，系统选择全部已发现技能。

当技能选择列表为空时，系统选择空集合。

当技能选择列表包含具体名称时，系统只选择名称完全匹配的技能。

不存在的技能名称不会产生选中项，也不会自动创建技能。

## 桌面端编辑

桌面端提供配置组页面，用户可以新增、删除、命名、描述和保存配置组。

桌面端展示当前工作区的技能列表，并允许配置组选择全部技能或部分技能。

保存配置组时，系统把配置写回工作区配置文件，并重新扫描工作区以更新页面状态。

当用户切换工作区或重新打开项目时，桌面端恢复该项目上次使用的配置组。

## 状态持久化

配置组本身持久化在工作区配置文件中。桌面端的当前配置组选择保存在本地界面状态中。

共享操作会把远端仓库地址、共享目标模式和项目名写入配置文件。配置组内容随配置文件一起参与共享。

## 校验与错误

系统查找配置组失败时返回配置组不存在错误。应用配置组和漂移检查都使用该错误语义。

系统保存配置时保留配置版本为 1，并规范化配置组名称和描述的前后空白。

## 成功标准

用户能够为一个 Skill 项目维护多组技能集合。

配置组能够稳定驱动应用目标、漂移检查和共享说明。

默认配置使新项目无需手动配置即可扫描和应用全部技能。

配置组保存后，桌面端和 CLI 读取到一致的配置结果。
