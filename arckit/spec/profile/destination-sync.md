# 目标应用与漂移

## 功能概述

目标应用与漂移功能把配置组中的技能和共享资产复制到目标目录，并检测来源与目标之间的文件差异。该功能支持用户把团队认可的技能集合安装到 agent 目录、项目目录或自定义目录。

## 目标类型

桌面端提供内置 agent 目标、项目目标和自定义目标三种选择。

内置 agent 目标包括当前用户主目录下的 Codex、Claude 和 Cursor 技能目录。

项目目标由用户通过目录选择器指定。自定义目标由用户手动输入路径。

CLI 通过目标目录参数指定应用或漂移检查的位置。缺省目标目录为工作区内的 `.skillops/skills`。

## 应用配置组

系统应用配置组时，先根据配置组选择技能，再解析目标目录为绝对路径。

系统确保目标目录存在。目标目录不存在时，系统创建目标目录。

每个选中技能被复制到目标目录下以技能名称命名的子目录。

目标目录中已存在同名技能目录时，系统跳过该技能，不覆盖已有目录。

系统把共享资产复制到目标目录下以资产名称命名的子目录。

目标目录中已存在同名共享资产目录时，系统跳过该资产，不覆盖已有目录。

应用结果包含配置组名称、目标目录、已复制技能、已跳过技能、已复制共享资产和已跳过共享资产。

## 目标历史

桌面端记录每个项目的应用目标历史。历史记录包含来源项目、配置组、目标名称、目标路径和最后应用时间。

同一项目、配置组和目标路径的记录会被最新应用结果替换。

桌面端在总览和应用目标页面展示最近的目标历史。

## 漂移检查

系统漂移检查使用同一配置组选中技能，并逐项比较来源目录和目标目录。

当目标技能目录不存在时，该技能状态为缺失。系统列出来源目录中的所有文件，并标记为缺失。

当目标技能目录存在时，系统比较来源和目标目录内所有文件的内容哈希。

来源存在而目标不存在的文件标记为缺失。

来源和目标都存在但哈希不同的文件标记为变更。

目标存在而来源不存在的文件标记为额外。

没有任何文件级差异时，该项状态为一致。存在任一文件级差异时，该项状态为变更。

共享资产使用与技能相同的漂移检查逻辑，并在漂移项中标记为资产类型。

## 漂移报告

漂移报告包含配置组名称、目标目录和漂移项列表。

每个漂移项包含名称、类型、状态、来源路径、目标路径、文件级差异和缺失、变更、额外数量汇总。

桌面端应用配置组后立即执行一次漂移检查，并展示复制与跳过数量。

桌面端支持打开完整漂移差异窗口。差异窗口按技能或资产展示状态、文件路径和来源、目标哈希摘要。

CLI 的漂移命令输出漂移报告 JSON。

## 不覆盖策略

应用配置组不会覆盖目标目录中已存在的同名技能或共享资产目录。

该策略使用户可以在目标侧保留本地修改，并通过漂移检查查看来源与目标的差异。

## 成功标准

用户能够把配置组选中的技能安装到 agent、项目或自定义目录。

系统能够同步共享资产并在结果中区分技能和资产。

系统能够以文件级差异说明目标目录是否落后、被修改或包含额外文件。

系统能够在不覆盖目标修改的前提下支持后续人工处理。
