# GitHub 共享

## 功能概述

GitHub 共享功能围绕 GitHub-first 工作流生成发布计划，并把 Skill 项目同步到指定 Git 仓库。该功能使个人或小团队可以用 Git 仓库作为技能共享来源，而不依赖托管注册中心。

## 来源地址解析

系统接受完整 Git URL、SSH Git URL、GitHub 仓库路径和 GitHub 页面路径。

用户输入 `owner/repo` 或 `github.com/owner/repo` 时，系统解析为 GitHub 仓库克隆地址。

用户输入 GitHub tree 或 blob URL 时，系统解析仓库地址、分支或标签，以及仓库内子目录。

用户输入非 GitHub 的 HTTP Git URL 时，系统按普通 Git URL 处理，不解析 GitHub 子目录语义。

来源地址为空或格式不合法时，系统返回错误。

## 下载共享来源

桌面端允许用户输入 GitHub 来源并下载 Skill 项目到本地缓存。

系统把来源仓库克隆到应用数据目录下的来源缓存。缓存目录名称由仓库名称和来源摘要组成。

当缓存目录已存在时，系统执行快进拉取以更新本地来源。

来源地址指定分支或标签时，系统在克隆或拉取后检出该引用。

来源地址指定子目录时，系统把该子目录作为 Skill 项目根目录返回。

子目录不存在时，系统返回错误。桌面端把该下载项目标记为失败并保留错误信息。

## 发布计划

发布计划基于当前工作区快照生成，不直接写入远端仓库。

计划包含工作区根目录、仓库名称、可见性、技能来源目录下的文件清单、安装命令和发布检查项。

仓库名称来自工作区根目录名称。安装引用优先使用配置中的团队仓库地址，缺省时使用占位 GitHub 仓库引用。

私有发布计划提示用户确认仓库权限和评审负责人。

公开发布计划提示用户移除私有 URL、内部路径和公司内部流程描述。

发布计划包含已发现技能数量、运行审计、补充 README、打标签发布等检查项。

## 共享目标路径

共享操作支持直接写入配置路径，也支持按项目名写入子目录。

直接写入模式使用来源地址中解析出的仓库子目录作为 Skill 项目根路径。

当直接写入模式的仓库子目录已经以当前 `sourceDir` 结尾时，系统把该子目录的父目录作为 Skill 项目根路径，避免生成重复的技能来源目录。例如来源地址为 `github.com/owner/repo/tree/main/definition/skills` 且 `sourceDir` 为 `skills` 时，系统写入 `definition/skills`，而不是 `definition/skills/skills`。

按项目名写入模式会把项目名追加到解析出的仓库子目录之后。

项目名为空且选择按项目名写入时，系统返回错误。

系统会清理目标路径中的空段、当前目录段和上级目录段，避免把项目写入非法相对路径。

## 共享执行

共享执行前，系统扫描当前工作区并准备共享仓库检出目录。

共享仓库检出目录位于应用数据目录下的共享工作树缓存。缓存目录名称由仓库名称和仓库地址摘要组成。

当检出目录不存在时，系统克隆目标仓库。检出目录存在时，系统拉取远端引用信息。

系统使用来源地址指定的分支；未指定分支时，系统优先使用远端默认分支，无法识别时使用 `main`。

远端分支存在时，系统基于远端分支创建或重置本地分支，并执行快进拉取。

远端分支不存在时，系统创建本地分支，并在推送时创建远端分支。

## 文件同步

系统在目标路径中创建目录，并替换目标路径下的技能来源目录。

系统复制当前工作区的技能来源目录到目标路径。

系统写入面向共享仓库的配置文件。该配置文件中的团队仓库地址指向可被安装的远端项目引用。

当前工作区存在 README 时，系统复制该 README 到目标路径。当前工作区没有 README 时，系统生成基础 README。

系统向 README 写入或更新 SkillOps 使用说明区块。该区块包含可见性、桌面端使用流程、配置组列表和 CLI 安装命令。

## 提交与推送

系统把目标路径变更加入 Git 暂存区。

当 Git 状态存在变更时，系统使用用户输入的提交信息创建提交。提交信息为空时，系统使用默认共享提交信息。

当 Git 状态没有变更时，系统不创建提交，并记录没有文件变更。

共享操作最后推送当前分支到远端，并返回远端地址、分支、目标路径、是否提交、是否推送和执行消息。

## 本地配置更新

共享前，系统把用户输入的远端仓库、共享目标模式和项目名保存到当前工作区配置。

本地配置中的团队仓库地址保留用户输入的远端来源。共享仓库中的配置使用可安装的远端项目引用。

## 成功标准

用户能够从 GitHub 来源下载一个 Skill 项目并作为本地项目打开。

用户能够生成私有或公开发布计划。

用户能够把当前 Skill 项目同步到指定 Git 仓库路径。

共享仓库中的 README 能说明桌面端使用流程、配置组和 CLI 安装命令。
