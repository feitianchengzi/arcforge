# 审计

## 功能概述

审计功能对工作区内的技能执行安全与质量检查。默认规则审计在本地同步完成；CLI 可选启用 Agent 辅助诊断，用于补充关键词规则难以覆盖的语义安全和治理边界风险。审计结果用于帮助用户在共享、发布或安装前发现密钥泄露、危险指令、元数据缺失和高风险 skill 行为问题。

## 审计范围

系统以已发现的技能列表作为审计输入。每个技能的 `SKILL.md` 是必审文件。

系统会继续读取技能目录下除 `SKILL.md` 之外的文本文件，并对这些文件执行密钥检查。

当文件读取失败或文件内容包含空字节时，系统不把该文件作为文本审计对象。

共享资产不作为独立技能执行完整结构审计。共享资产只有在归属于某个技能目录内部时才随文件遍历进入密钥检查。

## 能力边界

默认审计实现是本地规则扫描，不是完整安全审计。

规则集覆盖已知密钥模式、少量高风险指令短语、高风险 shell 命令模式，以及 `SKILL.md` 的基础元数据和结构问题。

Agent 或 hybrid 模式可以补充语义判断，但仍可能漏报或误报。用户在共享、发布或安装前仍需要人工复核技能内容。

桌面端和 CLI 输出都必须包含审计机制、能力边界说明和 GitHub issue 反馈地址。桌面端 issue 入口需要高亮展示并可点击跳转。

## 密钥检查

系统识别常见 OpenAI API Key 形式的密钥，并生成严重级发现。

系统识别常见 GitHub Token 形式的密钥，并生成严重级发现。

系统识别常见 Anthropic API Key、AWS Access Key、Slack Token 和 Slack Webhook 形式的密钥，并生成严重级发现。

系统识别私钥块起始标记，并生成严重级发现。

密钥发现包含规则代码、严重级别、说明、相对文件路径和命中行号。

## 危险指令检查

系统识别要求忽略既有指令、安全规则或系统规则的文本，并生成警告级发现。

系统识别要求读取、打开、打印或输出环境、凭据、SSH、AWS 或 npm 配置等敏感文件的文本，并生成警告级发现。

系统识别要求自动提交、推送或合并且不要求用户确认的文本，并生成警告级发现。

系统识别要求上传、发送或提交日志、密钥、环境变量、令牌或凭据的文本，并生成警告级发现。

系统识别常见破坏性 shell 命令、远程脚本管道执行、未审查执行用户输入代码，以及绕过审批、权限、沙箱或策略的文本，并生成警告级发现。

危险指令发现包含规则代码、严重级别、说明、相对文件路径和命中行号。

## 结构质量检查

系统检查 `SKILL.md` 是否包含 frontmatter。缺少 frontmatter 时生成警告级发现。

系统检查 frontmatter 是否包含稳定的 `name` 字段。缺少名称时生成警告级发现。

系统检查 frontmatter 是否包含清晰且长度足够的 `description` 字段。缺少描述或描述过短时生成警告级发现。

系统还根据技能摘要中的描述长度判断触发描述是否过短。描述缺失或长度不足时生成警告级发现。

系统检查 skill 写作质量。`description` 没有承担触发条件、适用范围、相邻 skill 分工或不触发场景时，系统生成警告级发现。

系统检查执行正文是否混入触发层补救说明。执行型 skill 正文大量解释“先用 A / 不用 B / 与 C 的区别”，而不是描述被触发后的正向执行流程时，系统生成警告级发现。

系统检查纠错泄漏。正文直接包含用户纠错、失败复盘、“不要再……”口吻或一次协作中的修正痕迹时，系统生成警告级发现。

系统检查业务耦合。通用 skill 把具体项目、客户、临时策略或一次性上下文写成普遍执行规则时，系统生成警告级发现。

写作质量检查不替代人工评审。系统将这些发现作为共享、正式化或发布准备前的质量风险提示，并建议回到 Skill First 更新目标 skill。

## Agent 辅助诊断

CLI 审计命令支持 `rule`、`agent` 和 `hybrid` 三种模式。`rule` 为默认模式；`agent` 只运行 Agent 辅助诊断；`hybrid` 合并规则审计和 Agent 诊断。

Agent 诊断使用固定审计 prompt 和临时技能副本，不直接在真实项目目录内执行诊断。默认适配器使用 Codex CLI 非交互模式，并要求只读沙箱、禁止审批请求、临时会话和超时控制。

Agent 诊断检查 prompt injection、忽略安全边界、读取或上传敏感信息、自动提交或发布、越权应用或共享、过宽触发描述、一次性上下文固化等语义风险。

Agent 诊断必须返回结构化发现。发现项包含来源、严重级别、说明、相对文件路径、可选行号、证据和置信度。Agent 不可用、超时或输出无效时，系统生成警告级发现并保留已有规则审计结果。

## 严重级别

审计发现支持信息、警告和严重三个级别。

严重级发现表示内容可能包含真实密钥或高风险泄露材料。用户在团队共享或公开发布前需要处理严重级发现。

警告级发现表示技能行为、元数据或安全边界存在不明确或高风险表述。

写作质量发现通常为警告级。若该表述会导致 agent 大概率误触发、错误写入或绕过确认，系统可以将其报告为更高风险的警告组合，并在说明中标注需要人工复核。

信息级发现用于低风险提示。当前规则集主要产生严重级和警告级发现。

## 审计评分

审计报告包含 0 到 100 的评分。系统从 100 分开始按发现扣分。

严重级发现扣分高于警告级发现。警告级发现扣分高于信息级发现。

评分最低为 0 分。评分只表达当前审计模式下的相对健康度，不表示技能绝对安全。

## 审计报告

审计报告包含工作区根路径、生成时间、技能列表、发现列表、评分、能力边界说明、GitHub issue 反馈地址、审计模式和可选 Agent 运行摘要。

桌面端在总览和审计页面展示审计分数、严重问题数量、警告数量、能力边界说明、GitHub issue 入口和发现列表。

CLI 的审计命令输出包含能力边界说明的审计报告 JSON。当存在严重级发现时，命令以严重问题退出码结束。

## 成功标准

系统能够在本地完成审计，不依赖远端服务。

系统能够给每条发现提供可定位的文件路径和可选行号。

系统能够区分高风险密钥问题和中风险质量问题。

系统能够为共享发布前的检查提供统一报告。

系统能够在 CLI 中按需启用 Agent 辅助诊断，并在 Agent 失败时给出结构化降级结果。
