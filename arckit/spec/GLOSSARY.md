# 术语表

## Skill 项目

Skill 项目是 SkillOps 管理的本地工作区。该工作区包含技能来源目录、配置文件和用于共享发布的项目元数据。

## 技能

技能是一个包含 `SKILL.md` 的目录。SkillOps 读取技能元数据、正文描述、引用资料目录和脚本目录，并将其作为可审计、可分组、可同步的资产。

## 共享资产

共享资产是技能来源目录下不包含 `SKILL.md` 的顶层目录。共享资产不作为独立技能展示，但在应用配置组时随目标一起复制。

## 配置组

配置组定义一组技能选择规则和目标 agent 标识。配置组支持包含全部技能，也支持显式列出指定技能。

## 目标目录

目标目录是配置组应用后的写入位置。目标目录可以是内置 agent 技能目录、另一个项目目录或用户输入的自定义目录。

## 漂移

漂移是来源目录与目标目录之间的文件差异。SkillOps 按文件哈希判断缺失、变更和额外文件，并汇总到技能或共享资产级别。

## 审计发现

审计发现是规则审计产生的风险或质量提示。每条发现包含严重级别、规则代码、说明、文件路径和可选行号。

## 审计评分

审计评分是根据发现扣分后的工作区健康值。严重问题、警告和信息级发现对应不同扣分权重，最低为 0 分。

## 发布计划

发布计划是 SkillOps 为 GitHub-first 共享生成的文件清单、安装命令和检查项。发布计划不直接修改远端仓库。

## 共享

共享是将当前 Skill 项目同步到指定 Git 仓库路径并提交推送的行为。共享会写入技能目录、配置文件和 README 使用说明。

## 来源地址

来源地址是用户输入的 GitHub 或 Git URL。SkillOps 支持仓库根路径，也支持 GitHub tree 或 blob URL 中的分支和子目录。

## 工作区快照

工作区快照是一次扫描的结果，包含根路径、配置、技能列表、共享资产列表和审计报告。
