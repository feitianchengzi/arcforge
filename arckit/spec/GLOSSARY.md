# 术语表

## Skill 项目

Skill 项目是 ArcForge 管理的本地工作区。该工作区包含技能来源目录、配置文件和用于共享发布的项目元数据。

## 技能

技能是一个包含 `SKILL.md` 的目录。ArcForge 读取技能元数据、正文描述、引用资料目录和脚本目录，并将其作为可审计、可分组、可同步的资产。

## 能力入口

能力入口是 agent 面向一类用户任务选择的 skill。用户任务可以只需要一个能力入口，也可以需要多个 skill 分工协作。CLI、server、UI、MCP、脚本和内部服务属于能力入口背后的实现承载，不直接替代 skill 入口。

## 软件能力单元

软件能力单元是以 skill 为交互入口，并按需组合 `SKILL.md`、引用文件、CLI、server、UI、MCP、脚本、状态、schema、测试 fixture 和回传机制的能力包。软件能力单元可以保持轻量，也可以随着真实任务验证扩展实现承载。

## 实现承载

实现承载是目标 skill 背后用于执行、呈现、保存状态或回传结果的具体能力。实现承载可以存在于当前项目、其他仓库、CLI、server、Desktop UI、Web UI、MCP、内部服务、schema、fixture 或脚本中。

## Skill First

Skill First 是创建、更新和验证目标 skill 的前段工作方式。它先定位目标 skill 来源和工作副本，再基于真实任务和子代理反馈迭代 skill、引用文件或实现承载。Skill First 验证完成后只建议进入 ArcForge 治理阶段，不自动执行治理写入。

## 共享资产

共享资产是技能来源目录下不包含 `SKILL.md` 的顶层目录。共享资产不作为独立技能展示，但在应用配置组时随目标一起复制。

## 配置组

配置组定义一组技能选择规则和目标 agent 标识。配置组支持包含全部技能，也支持显式列出指定技能。

## 目标目录

目标目录是配置组应用后的写入位置。目标目录可以是内置 agent 技能目录、另一个项目目录或用户输入的自定义目录。

## 漂移

漂移是来源目录与目标目录之间的文件差异。ArcForge 按文件哈希判断缺失、变更和额外文件，并汇总到技能或共享资产级别。

## 审计发现

审计发现是规则审计产生的风险或质量提示。每条发现包含严重级别、规则代码、说明、文件路径和可选行号。

## 审计评分

审计评分是根据发现扣分后的工作区健康值。严重问题、警告和信息级发现对应不同扣分权重，最低为 0 分。

## 发布计划

发布计划是 ArcForge 为 GitHub-first 共享生成的文件清单、安装命令和检查项。发布计划不直接修改远端仓库。

## 共享

共享是将当前 Skill 项目同步到指定 Git 仓库路径并提交推送的行为。共享会写入技能目录、配置文件和 README 使用说明。

## 来源地址

来源地址是用户输入的 GitHub 或 Git URL。ArcForge 支持仓库根路径，也支持 GitHub tree 或 blob URL 中的分支和子目录。

## 工作区快照

工作区快照是一次扫描的结果，包含根路径、配置、技能列表、共享资产列表和审计报告。
