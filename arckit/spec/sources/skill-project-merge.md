# Skill 项目归并与应用关系

## 功能概述

该能力把一个 Skill 项目直接作为另一个项目的来源使用，不再设计独立的来源清单或来源标记。用户可以把当前项目中自然产生的技能归并到另一个 Skill 项目的指定目录；归并成功后，当前项目保存一条应用关系，用于后续漂移检查和重新应用。

远程 Git 或 GitHub 地址只是 Skill 项目的获取方式。系统先把远程地址下载到本地缓存，再以本地项目目录参与归并、应用和漂移。

Git 更新检查是独立能力。它只检查当前工作区是否是带上游的 Git checkout，不依赖归并关系、应用关系或来源登记。

## 核心概念

Skill 项目是包含 SkillOps 配置、技能目录和共享资产的本地目录。任何可扫描的 Skill 项目都可以成为归并目标或应用来源。

归并是把当前项目中的一个或多个技能复制到另一个 Skill 项目的目标目录，并合并目标项目配置组。

应用关系描述当前项目消费哪个来源 Skill 项目、哪个配置组、哪些技能以及应用到哪个目标目录。应用关系只保存在当前项目的用户级状态中。

远程来源是一次性解析输入，不是持久实体。用户输入远程地址时，系统下载或复用本地缓存 checkout，并把解析后的本地目录写入应用关系。

## Skill 项目解析

系统支持本地目录、完整 Git URL、SSH Git URL、GitHub 仓库路径、GitHub 页面路径以及带分支或子目录的 GitHub tree/blob URL。

本地目录会解析为绝对路径并通过扫描确认 Skill 项目结构。远程地址会下载到用户级缓存目录；缓存已存在时系统复用 checkout 并获取远端引用。

解析结果只返回本地项目根目录。系统不创建全局来源记录，不要求用户给来源命名，也不为项目写入额外来源标记。

## 归并计划

归并计划读取当前项目快照、目标 Skill 项目快照和用户选择的技能，计算写入目标项目后的文件变化。计划不写入文件。

用户必须指定归并目标项目和目标路径。目标路径是相对目标项目根目录的路径。系统清理空段、当前目录段和上级目录段，并拒绝指向目标项目外部的路径。

系统逐项比较当前项目技能目录和目标项目目标目录中的现有目录。目标不存在时，该技能状态为新增。目标存在且内容一致时，状态为一致。目标存在且内容不同或目标路径包含非目录文件时，状态为冲突。

归并计划包含目标项目根目录、目标项目名称、配置组、目标路径、当前项目应用目标目录、技能计划项、应用关系草案和冲突标记。

## 归并执行

归并执行必须显式确认。

存在冲突时，系统停止执行并返回冲突清单，不覆盖目标项目中的不同内容。

写入目标项目时，系统按技能目录执行临时目录复制和原子替换。复制失败时保留原目标内容。

系统在目标项目配置中合并指定配置组。配置组不存在时创建配置组；配置组存在时追加本次归并的技能名称并保留已有技能、描述和目标。

归并成功后，系统在当前项目记录应用关系。当前项目已有同来源根目录、同配置组和同目标目录的应用关系时，系统合并技能列表，不覆盖无关应用关系。

归并不自动提交或推送目标项目 Git 变更。用户可以使用 Git 或共享交付功能处理提交、推送和 Pull Request。

## 应用关系

应用关系保存到用户级项目状态。应用关系包含来源 Skill 项目根目录、可选远程地址、来源名称、配置组、目标目录、技能列表、可选提交哈希、上次应用时间和更新时间。

系统支持列出、删除和手动添加当前项目的应用关系。

系统支持基于应用关系执行漂移检查。漂移检查以来源 Skill 项目作为来源，以当前项目记录的目标目录作为目标，输出文件级缺失、变更和额外文件。

系统支持基于应用关系重新应用。重新应用使用来源 Skill 项目配置组选择的技能和共享资产，并复制到当前项目记录的目标目录。

## 一次性应用与漂移

系统支持从指定 Skill 项目一次性应用配置组到目标目录。一次性应用不创建应用关系，除非用户显式要求保存。

系统支持从指定 Skill 项目一次性检查目标目录漂移。一次性漂移不创建应用关系。

用户未指定来源 Skill 项目时，一次性应用和漂移使用当前项目作为来源。

## Git 更新检查

Git 更新检查只面向当前工作区根目录。系统检查当前目录是否有 Git remote、当前分支是否能解析上游、工作区是否 dirty，以及本地相对上游的 ahead/behind 数量。

桌面端总览页支持自动检查。当前项目存在 Git remote，且上次检查超过 24 小时或从未检查时，系统自动执行一次状态检查并保存结果。自动检查不执行更新、不弹确认，也不覆盖用户本地变更。

Git 更新执行必须显式确认。更新只执行 fast-forward-only pull。当本地存在未提交变更、本地领先上游、处于 detached HEAD 或无法解析上游分支时，系统停止更新并返回错误。

该能力在桌面端总览展示，在 CLI 中通过 `source status` 和 `source update` 暴露。

## 边界情况

来源或目标项目不可访问时，系统返回文件系统错误，不创建或删除应用关系。

目标项目没有技能时，归并仍可创建目标目录和配置组。

选择的技能在当前项目不存在时，系统返回明确错误。

目标路径位于目标项目根目录之外时，系统拒绝归并。

远程地址无法下载、分支不存在或子目录不存在时，系统返回来源解析错误。

## 成功标准

用户能够直接选择本地目录或远程仓库作为目标 Skill 项目，不需要先登记来源。

用户能够把当前项目中的技能归并到目标 Skill 项目目录。

归并执行能够合并目标项目配置组，并在当前项目中生成应用关系。

当前项目能够基于应用关系检查漂移和重新应用来源 Skill 项目中的正式版本。

用户能够在总览中独立检查和更新当前 Git checkout。
