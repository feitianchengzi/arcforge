# 工作区发现

## 功能概述

工作区发现功能把一个本地目录识别为 ArcForge 可管理的 Skill 项目。该功能读取项目配置，扫描技能来源目录，识别技能和共享资产，并生成包含审计结果的工作区快照。

## 适用用户

该功能服务于维护个人或团队技能仓库的用户。用户可以打开一个已有项目，也可以初始化一个新的 ArcForge 配置。

## 工作区配置

ArcForge 把来源仓库内容和本地用户状态分开持久化。日常配置组、应用目标、共享目标、团队仓库偏好和来源目录覆盖值保存到用户级项目状态目录 `~/.arcforge/projects`。桌面端和 CLI 共同使用该存储位置，因此基础能力不依赖 Electron 专属状态文件。

项目根目录下的 `arcforge.config.json` 不作为持久配置保留。系统发现该文件时执行迁移：如果用户级项目状态不存在，系统先把文件内容写入 `~/.arcforge/projects`；如果用户级项目状态已经存在，系统不覆盖用户级状态。两种情况下系统都会删除项目根目录下的 `arcforge.config.json`。

当工作区没有用户级项目状态时，系统使用默认配置生成扫描上下文。默认情况下系统使用 `skills` 作为技能来源目录；如果工作区根目录本身包含 `SKILL.md` 或大小写不同的 `skill.md`，系统把根目录视为单个技能来源并使用 `.` 作为技能来源目录。默认配置提供名为 `default` 的配置组。

初始化工作区时，系统创建或刷新用户级项目状态，不向来源 checkout 写入 `arcforge.config.json`。

系统保存配置时保持版本为 1，并保留配置组的名称、描述、技能选择、目标标识和目标组合列表。

## 技能发现

系统从配置指定的来源目录开始递归扫描。任一目录中存在 `SKILL.md` 或大小写不同的 `skill.md` 时，该目录被识别为一个技能。

技能名称优先来自 `SKILL.md` frontmatter 中的 `name` 字段。缺少名称时，系统使用技能目录名作为名称。

技能描述优先来自 frontmatter 中的 `description` 字段。缺少描述时，系统从正文中第一个非标题段落提取摘要。

技能目标来自 frontmatter 中的 `metadata.targets` 或 `targets` 字段。系统只接收字符串数组形式的目标值。

技能摘要包含名称、描述、绝对路径、相对路径、目标列表、可选版本、是否存在引用资料目录和是否存在脚本目录。

扫描结果按技能名称排序。不存在来源目录时，技能列表为空。

## 共享资产发现

系统把来源目录下不包含任何 `SKILL.md` 或大小写不同的 `skill.md` 的顶层目录识别为共享资产。来源目录本身是一个技能目录时，系统不把它的 `references`、`scripts` 等子目录识别为共享资产。

共享资产用于保存多个技能共同依赖的资料、脚本或模板。共享资产不参与技能审计的技能级统计，但会在配置组应用时同步到目标目录。

系统忽略 `.git` 和 `node_modules` 目录。共享资产结果按名称排序。

## 工作区快照

扫描工作区时，系统依次加载配置、发现技能、发现共享资产并执行审计。

工作区快照包含根目录、配置、技能列表、共享资产列表和审计报告。桌面端和 CLI 使用同一工作区快照作为后续页面展示、发布计划、漂移检查和配置组应用的输入。

## GitHub 来源维护

当工作区来自 GitHub 或其他 Git 远端时，系统把本地 checkout 视为用户可检查、可决定是否更新的来源副本。

系统支持读取当前分支、上游分支、远端地址、本地 HEAD、上游 HEAD、ahead commit 数、behind commit 数、本次检查前的上次 fetch 时间和工作区是否存在未提交变更。

更新检查会获取上游 refs，并报告本地相对上游领先或落后的 commit 数。该检查不合并远端内容，也不修改工作区文件。

来源更新必须由用户显式确认。系统只执行 fast-forward 更新；当本地存在未提交变更、当前分支领先上游或无法解析上游分支时，系统停止更新并返回错误。来源检查前系统会先执行项目配置迁移与删除；迁移删除造成的 Git 状态按普通工作区变更处理。

桌面端可以展示来源状态和更新按钮，但基础能力由 CLI 和共享 core 提供。

## 异常与空状态

来源目录不存在时，系统返回空技能列表和空共享资产列表，扫描流程仍生成快照。

配置文件存在但 JSON 无法解析时，系统将错误返回给调用方。桌面端以状态信息展示错误，CLI 以错误输出结束命令。

## 成功标准

系统能够在本地目录中稳定识别所有包含 `SKILL.md` 或大小写不同的 `skill.md` 的技能。

系统能够在没有配置文件的目录中给出默认扫描行为。

系统能够区分技能目录和共享资产目录。

系统能够为桌面端和 CLI 提供一致的工作区快照。
