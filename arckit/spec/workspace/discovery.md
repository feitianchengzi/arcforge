# 工作区发现

## 功能概述

工作区发现功能把一个本地目录识别为 SkillOps 可管理的 Skill 项目。该功能读取项目配置，扫描技能来源目录，识别技能和共享资产，并生成包含审计结果的工作区快照。

## 适用用户

该功能服务于维护个人或团队技能仓库的用户。用户可以打开一个已有项目，也可以初始化一个新的 SkillOps 配置。

## 工作区配置

SkillOps 使用工作区根目录下的 `skillops.config.json` 作为控制文件。配置文件记录配置版本、技能来源目录、团队仓库地址、共享目标模式、共享项目名和配置组列表。

当工作区没有配置文件时，系统使用默认配置生成扫描上下文。默认情况下系统使用 `skills` 作为技能来源目录；如果工作区根目录本身包含 `SKILL.md` 或大小写不同的 `skill.md`，系统把根目录视为单个技能来源并使用 `.` 作为技能来源目录。默认配置提供名为 `default` 的配置组。

初始化工作区时，系统在目标根目录写入配置文件。初始化后的配置版本为 1，默认来源目录通常为 `skills`；如果目标根目录本身包含技能文件，则来源目录为 `.`。默认配置组包含全部技能并面向 Codex、Claude 和 Cursor。

系统保存配置时保持版本为 1，并保留配置组的名称、描述、技能选择和目标标识。

## 技能发现

系统从配置指定的来源目录开始递归扫描。任一目录中存在 `SKILL.md` 或大小写不同的 `skill.md` 时，该目录被识别为一个技能。

技能名称优先来自 `SKILL.md` frontmatter 中的 `name` 字段。缺少名称时，系统使用技能目录名作为名称。

技能描述优先来自 frontmatter 中的 `description` 字段。缺少描述时，系统从正文中第一个非标题段落提取摘要。

技能目标来自 frontmatter 中的 `metadata.targets` 或 `targets` 字段。系统只接收字符串数组形式的目标值。

技能摘要包含名称、描述、绝对路径、相对路径、目标列表、可选版本、是否存在引用资料目录和是否存在脚本目录。

扫描结果按技能名称排序。不存在来源目录时，技能列表为空。

## 共享资产发现

系统把来源目录下不包含任何 `SKILL.md` 或大小写不同的 `skill.md` 的顶层目录识别为共享资产。来源目录本身是一个技能目录时，系统不把它的 `references`、`scripts` 等子目录识别为共享资产。

共享资产用于保存多个技能共同依赖的资料、脚本或模板。共享资产不参与技能审计的技能级统计，但会在配置组应用时同步到目标目录。

系统忽略 `.git` 和 `node_modules` 目录。共享资产结果按名称排序。

## 工作区快照

扫描工作区时，系统依次加载配置、发现技能、发现共享资产并执行审计。

工作区快照包含根目录、配置、技能列表、共享资产列表和审计报告。桌面端和 CLI 使用同一工作区快照作为后续页面展示、发布计划、漂移检查和配置组应用的输入。

## 异常与空状态

来源目录不存在时，系统返回空技能列表和空共享资产列表，扫描流程仍生成快照。

配置文件存在但 JSON 无法解析时，系统将错误返回给调用方。桌面端以状态信息展示错误，CLI 以错误输出结束命令。

## 成功标准

系统能够在本地目录中稳定识别所有包含 `SKILL.md` 或大小写不同的 `skill.md` 的技能。

系统能够在没有配置文件的目录中给出默认扫描行为。

系统能够区分技能目录和共享资产目录。

系统能够为桌面端和 CLI 提供一致的工作区快照。
