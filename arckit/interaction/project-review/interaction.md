# Project Review - 交互规范

## 交互策略

### 核心任务

用户在项目审阅流程中判断当前 Skill 项目的健康度，检查 GitHub 来源是否落后上游，定位并编辑技能文件、共享资产和审计发现，并进入后续配置、应用或共享任务。

### 主路径

系统扫描成功后展示总览指标和来源状态入口。用户先查看技能数量、共享资产数量、审计评分、风险数量和 GitHub 来源更新状态，再进入技能文件编辑、审计、配置组、目标应用或共享页面。

### 决策点

用户根据严重问题数量、警告数量、技能文件内容完整性、来源是否落后上游和目标历史判断项目是否可以继续安装或共享。系统提供可编辑文件、文件路径、行号、来源 ahead/behind 数和上次 fetch 时间作为判断依据。

### 信息揭示

总览先展示健康指标、来源状态和下一步任务。技能页展示 `sourceDir` 文件树、配置组过滤器和当前文件编辑器。审计页优先展示评分与风险计数，再展示可定位发现。

### 状态流

进入页面时沿用应用外壳的扫描状态。扫描成功后进入内容状态。GitHub 来源项目显示来源状态入口，非 GitHub 来源项目不显示该入口。GitHub 来源状态初始为未检查。用户触发检查后，系统显示检查中、检查结果或错误状态。技能目录为空或配置组过滤后没有文件时展示空状态。审计发现为空时展示无发现状态。扫描失败时由外壳展示错误，审阅区域保留可恢复说明。

### 反馈与恢复

用户在技能编辑器保存文件后可以通过重新扫描刷新审计结果。用户检查 GitHub 来源后可以根据落后 commit 数自主选择是否更新；系统只在可快进且用户确认后更新来源。没有可展示文件或没有审计发现时，页面显示明确空状态而不是隐藏区域。

### 输入输出边界

输入为工作区快照、来源状态检查操作、来源更新确认、页签切换、配置组过滤、文件选择和文件编辑。输出为指标卡、来源更新状态、文件树、文件内容、审计发现列表和下一步入口。技能编辑器通过受限 IPC 写入当前工作区内的文本文件，来源维护通过共享 CLI/core 能力执行。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 加载中 | 项目扫描中 | ProgressView |
| 成功（内容） | 快照存在且数据可展示 | Tree / TextEditor / ScrollView / TabView / SourceStatus |
| 来源检查中 | 用户点击检查来源更新 | ProgressView / SourceStatus |
| 来源可更新 | GitHub 来源落后上游且可快进 | SourceStatus / Confirmation |
| 空状态 | 技能文件、共享资产或目标历史为空 | ContentUnavailableView |
| 错误 | 扫描失败或审计读取失败 | topbar error status + stage error state |

## 导航流程

### 进入方式

应用外壳扫描项目成功后进入项目审阅总览。

### 页面内导航

用户通过页签查看 Overview、Skills 和 Audit，也可以通过下一步入口跳转到 Profiles、Destinations 或 Share。Skills 页签内部通过文件树切换当前文件。来源状态入口保留在 Overview 中，不切换到 Share 页。

### 离开方式

用户切换到配置组、目标应用、共享页面或切换项目。

## 交互行为

### 总览指标

- **查看指标:** 系统展示技能数量、共享资产数量、审计评分、严重问题和警告数量。
- **点击下一步入口:** 系统切换到对应任务页签。

### 来源状态

- **未检查:** GitHub 来源项目在总览显示检查入口、当前分支和远端摘要；本地来源项目不显示来源维护卡。
- **检查更新:** 用户点击检查入口后，系统调用来源状态检查并显示检查中状态。
- **展示结果:** 系统展示 ahead commit 数、behind commit 数、上次 fetch 时间、检查时间和是否可更新。
- **确认更新:** 当 `canUpdate` 为真时，系统显示更新入口。用户点击后，系统执行确认更新并在完成后重新扫描工作区。
- **阻止更新:** 当本地有未提交变更、本地领先上游、没有上游或检查失败时，系统展示原因并不显示确认更新入口。

### 技能文件编辑

- **切换配置组过滤:** 系统按配置组包含的技能过滤文件树，并保留共享资产目录。
- **展开或收起目录:** 系统在文件树内展开或收起父子文件夹。
- **选择文件:** 系统读取当前工作区内的文本文件并展示到编辑器。
- **编辑文件:** 系统在编辑器内保留未保存内容，并提示未保存状态。
- **保存文件:** 系统通过受限 IPC 写入当前文件，并保留当前文件选择。
- **重载文件:** 系统重新读取磁盘内容并替换编辑器草稿。
- **打开独立窗口:** 系统打开与内置编辑器一致的独立窗口，并传递当前文件、配置组、目录收起状态和滚动位置。

### 审计发现

- **查看发现:** 系统展示严重级别、规则代码、说明、文件路径和可选行号。
- **重新扫描:** 系统刷新发现列表并同步总览计数。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| Skill File Editor | 点击独立打开技能或文件 | Window | 文件树、配置组过滤、当前文件编辑器、保存和重载 |
| Source Update Confirmation | 点击可更新来源的更新入口 | Confirmation | 落后 commit 数、fast-forward-only 说明、确认与取消 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 技能文件为空 | 文件树空状态 | 检查 sourceDir 或配置组后重新扫描 |
| 文件不可编辑 | 编辑器错误状态 | 选择文本文件或检查文件大小 |
| 来源无上游 | 来源状态错误 | 在 Git 中设置 upstream 或重新下载来源 |
| 来源有未提交变更 | 来源状态错误 | 提交、暂存或放弃本地变更后重新检查 |
| 来源领先上游 | 来源状态错误 | 使用 Git 手动处理本地提交 |
| 来源检查失败 | 来源状态错误 | 检查网络、远端权限和 Git 状态后重试 |
| 共享资产为空 | 资产区域空状态 | 继续其他任务 |
| 审计发现为空 | 审计区域空状态 | 继续配置或共享 |
| 扫描失败 | 外壳顶部错误状态 | 重新扫描或切换项目 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 初次扫描 | 使用外壳顶部扫描状态 |
| 重新扫描 | 保留当前审阅页签，完成后替换数据 |
| 大量发现 | 列表滚动展示，不压缩路径和行号 |
| 大量文件 | 文件树和编辑器分别滚动，不带动项目摘要和阶段入口滚动 |
