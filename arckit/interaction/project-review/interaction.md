# Project Review - 交互规范

## 交互策略

### 核心任务

用户在项目审阅流程中判断当前 Skill 项目的健康度，定位技能、共享资产和审计发现，并进入后续配置、应用或共享任务。

### 主路径

系统扫描成功后展示总览指标。用户先查看技能数量、共享资产数量、审计评分和风险数量，再通过下一步入口进入审计、配置组、目标应用或共享页面。

### 决策点

用户根据严重问题数量、警告数量、技能列表完整性和目标历史判断项目是否可以继续安装或共享。系统提供文件路径和行号作为修复依据。

### 信息揭示

总览先展示健康指标和下一步任务。技能页展示技能名称、描述和相对路径。审计页优先展示评分与风险计数，再展示可定位发现。

### 状态流

进入页面时沿用应用外壳的扫描状态。扫描成功后进入内容状态。技能或共享资产为空时展示空状态。审计发现为空时展示无发现状态。扫描失败时由外壳展示错误，审阅区域保留可恢复说明。

### 反馈与恢复

用户修复技能文件后通过重新扫描刷新审计结果。没有共享资产或没有审计发现时，页面显示明确空状态而不是隐藏区域。

### 输入输出边界

输入为工作区快照和页签切换。输出为指标卡、技能列表、共享资产列表、审计发现列表和下一步入口。审阅流程不写入文件。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 加载中 | 项目扫描中 | ProgressView |
| 成功（内容） | 快照存在且数据可展示 | List / ScrollView / TabView |
| 空状态 | 技能、共享资产或目标历史为空 | ContentUnavailableView |
| 错误 | 扫描失败或审计读取失败 | ContentUnavailableView + toolbar |

## 导航流程

### 进入方式

应用外壳扫描项目成功后进入项目审阅总览。

### 页面内导航

用户通过页签查看 Overview、Skills 和 Audit，也可以通过下一步入口跳转到 Profiles、Destinations 或 Share。

### 离开方式

用户切换到配置组、目标应用、共享页面或切换项目。

## 交互行为

### 总览指标

- **查看指标:** 系统展示技能数量、共享资产数量、审计评分、严重问题和警告数量。
- **点击下一步入口:** 系统切换到对应任务页签。

### 技能列表

- **查看技能:** 系统展示技能名称、描述和相对路径。
- **查看共享资产:** 系统展示资产名称和相对路径。

### 审计发现

- **查看发现:** 系统展示严重级别、规则代码、说明、文件路径和可选行号。
- **重新扫描:** 系统刷新发现列表并同步总览计数。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| 无 | 审阅流程不使用弹窗 | - | 审阅流程只展示页面内状态 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 技能列表为空 | 技能区域空状态 | 检查 sourceDir 后重新扫描 |
| 共享资产为空 | 资产区域空状态 | 继续其他任务 |
| 审计发现为空 | 审计区域空状态 | 继续配置或共享 |
| 扫描失败 | 外壳顶部错误状态 | 重新扫描或切换项目 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 初次扫描 | 使用外壳顶部扫描状态 |
| 重新扫描 | 保留当前审阅页签，完成后替换数据 |
| 大量发现 | 列表滚动展示，不压缩路径和行号 |
