# Share - 交互规范

## 交互策略

### 核心任务

用户在 Share 页面把当前 Skill 项目中的技能整理为团队共享或发布前检查材料。

### 主路径

页面展示共享范围、目标远端、发布准备项、审计结果和执行计划。用户选择共享范围和远端后生成计划，确认后系统执行本地提交辅助、远端提示或发布检查清单。

### 决策点

用户决定共享当前项目或某个配置组；决定目标分支、提交信息和是否准备 PR。系统在执行前展示缺失项、权限状态和即将写入的文件。

### 信息揭示

页面先展示共享范围和状态，再展示发布准备检查，最后展示执行计划和结果。GitHub 或 ClawHub/OpenClaw 只作为发布准备目标，不作为 ArcForge 内置市场。

### 状态流

无可共享内容时显示空状态。选择范围后生成检查清单。存在阻断项时禁用执行。确认后显示执行进度。完成后展示下一步命令或链接提示。

### 反馈与恢复

执行失败保留范围、远端和提交信息。权限失败保留计划并提示用户在外部授权。共享完成后可返回 Audit 或 Skills 修复后再次生成计划。

### 输入输出边界

输入包括共享范围、远端、分支、提交信息、PR 选项和确认。输出为发布检查清单、执行日志、命令提示和结果摘要。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 范围选择 | 进入 Share 页面 | ScopePicker |
| 检查清单 | 用户选择范围 | Checklist |
| 执行确认 | 检查通过后点击 Share | ConfirmationDialog |
| 执行中 | 用户确认执行 | ProgressView |
| 完成/失败 | 执行结束 | ResultView |

## 交互行为

- **选择范围:** 用户选择当前项目或配置组。
- **生成计划:** 系统生成共享文件、检查项和外部命令提示。
- **检查阻断:** 阻断项存在时执行按钮不可用。
- **确认执行:** 用户确认后系统执行本地写入或 Git 辅助动作。
- **查看结果:** 完成后展示下一步 GitHub 或 OpenClaw 操作提示。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| Share confirmation | 点击 Share | ConfirmationDialog | 范围、远端、文件和提交信息 |
| Permission issue | 权限不足 | Alert | 缺失权限和外部修复方式 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 缺少远端 | 检查项阻断 | 配置远端 |
| 审计阻断 | 检查项阻断 | 前往 Audit |
| Git 执行失败 | 结果错误 | 修正后重试 |
| 权限不足 | Alert | 外部授权后重试 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 计划生成 | 保留范围选择并显示检查中 |
| 执行共享 | 显示步骤日志 |
| 完成后刷新 | 刷新检查清单和结果摘要 |
