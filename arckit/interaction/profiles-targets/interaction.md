# 配置组与目标交互

## 策略基线

配置组与目标流程把“选择哪些技能”和“安装到哪里”拆成相邻但独立的任务。配置组编辑采用草稿模型，目标应用采用先确认目标路径再执行的模型，漂移检查提供执行前后恢复依据。

## 策略锚点

- 配置组编辑在保存前不改变已持久化配置。
- 目标路径必须在应用和漂移检查前明确显示。
- 应用操作不覆盖目标中已存在的同名目录。
- 应用完成后立即展示复制、跳过和漂移结果。
- 漂移差异必须能打开完整文件级窗口。

## 状态机

### 配置组列表状态

配置组页面左侧展示配置组列表。当前选中的配置组在右侧进入编辑状态。

用户新增配置组时，系统创建唯一名称的草稿项并选中。用户删除配置组时，至少保留一个配置组。

### 配置组草稿状态

右侧表单展示名称、描述、技能选择和目标 agent 标识。用户编辑时只修改草稿。

选择全部技能时，单项技能复选框不可编辑。取消全部技能后，用户可以逐项选择技能。

保存后，系统写入配置、重新扫描并保持当前配置组选择。取消后，系统恢复为来源配置。

### 目标选择状态

应用目标页右侧展示来源项目、配置组选择、选中技能数量、目标类型和目标路径。

目标类型使用分段控件。agent 目标展示默认 agent 目录选择器，项目目标展示目录输入与目录选择按钮，自定义目标展示路径输入。

目标路径为空时，应用和漂移检查按钮不可执行。

### 应用执行状态

用户点击应用后，系统复制选中技能和共享资产。执行期间保留表单内容和目标路径。

应用完成后，系统展示已复制和已跳过数量，并立即触发漂移检查。跳过项作为正常结果展示。

应用失败时，系统保留目标路径和配置组选择，顶部状态或页面状态展示错误，用户可以重试或改目标。

### 漂移检查状态

用户可以在应用前或应用后执行漂移检查。未执行时，漂移区域展示空状态。

漂移报告按技能和共享资产展示状态。每项展示缺失、变更和额外文件数量。

状态全部一致时，列表仍展示一致状态，证明目标可追踪。

### 完整差异窗口

用户点击查看完整差异后打开独立差异窗口。窗口展示配置组、目标目录、总数、变更数、一致数和每个条目的文件级差异。

差异窗口不修改项目。关闭窗口后返回应用目标页，保留当前漂移报告。

## 触发与反馈

新建配置组按钮添加草稿配置组。删除配置组按钮在只剩一个配置组时不可用。

保存配置组按钮触发配置保存和扫描。保存完成后，状态显示配置已保存。

检查漂移按钮触发漂移报告。应用按钮触发复制并随后刷新漂移报告。

选择项目目标按钮打开目录选择器。取消选择时保留原目标路径。

查看完整差异按钮仅在已有漂移报告时可用。

## 容错与恢复

配置保存失败时，草稿内容仍保留在页面上，用户可以修正后再次保存。

目标目录不存在时，应用操作由系统创建目标目录。漂移检查对不存在目标返回缺失状态。

目标目录已有同名技能或资产时，应用跳过而不是覆盖。用户通过漂移结果判断是否需要人工处理。

目录选择取消不清空用户已输入路径。

## 信息优先级

配置组页面优先展示配置组列表与当前草稿表单，技能选择和目标 agent 选择作为表单细节。

应用目标页优先展示目标历史和漂移结果，因为这些信息决定是否需要再次应用。当前应用表单紧邻主操作，保证执行前可确认来源、配置组和目标。

漂移列表优先展示状态和差异计数，文件级哈希进入完整差异窗口。

## 平台语义

配置组列表对应 `List(sidebar)`，编辑表单对应 `Form`。目标类型对应 `Picker(segmented)`。漂移结果对应 `List`，完整差异窗口对应独立 macOS `Window`，文件差异对应 `Table`。
