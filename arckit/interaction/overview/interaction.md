# Overview - 交互规范

## 交互策略

### 核心任务

用户在 Overview 页面快速判断当前项目状态，并进入技能编辑与归并、配置组、应用目标、共享发布或审计页面。

### 主路径

系统完成项目扫描后进入 Overview。页面展示项目摘要、技能数量、共享资产数量、审计评分、严重问题数量、警告数量、当前 Git checkout 更新状态、最近应用历史和下一步任务。用户通过页面入口进入对应页面处理具体问题。

### 决策点

用户根据健康指标决定先修复审计问题、归并 Skill、调整配置组、检查应用目标、检查 Git 更新或准备共享。系统展示摘要和入口；Git 更新必须由用户显式确认后才执行。

### 信息揭示

页面先展示项目身份和核心指标，再展示需要处理的事项，最后展示页面导航入口。异常信息以摘要方式出现，并链接到具体页面。

### 状态流

扫描中显示保留上下文的加载态。扫描成功显示总览。无技能时显示空状态和创建入口。扫描失败时保留项目路径、错误原因和重试入口。

### 反馈与恢复

重新扫描保留当前页面并刷新摘要。进入子页面前不丢失当前项目上下文。扫描失败允许重试或返回 Home。

### 输入输出边界

输入包括重新扫描、页面入口点击和项目返回。输出为扫描摘要、状态入口和错误提示。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 加载 | Home 打开项目或用户重新扫描 | ProgressView |
| 总览 | 扫描成功且存在技能 | Dashboard / NavigationGrid |
| 空项目 | 扫描成功但无技能 | ContentUnavailableView |
| 错误 | 扫描失败 | InlineError |

## 交互行为

- **重新扫描:** 用户点击 Rescan 后系统刷新项目快照和总览指标。
- **进入技能:** 用户点击 Skills 入口后查看、编辑或归并 Skill。
- **进入配置组:** 用户点击 Profiles 入口后编辑项目配置组。
- **进入应用目标:** 用户点击 Destinations 入口后查看应用状态与漂移。
- **进入共享:** 用户点击 Share 入口后准备发布或团队共享。
- **进入审计:** 用户点击 Audit 入口后查看具体发现。
- **自动检查更新:** 当前项目存在 Git remote 时，进入 Overview 会在上次检查超过 24 小时或无历史检查时自动检查当前 checkout 相对上游的 ahead/behind 状态。
- **手动检查更新:** 用户可随时点击检查按钮刷新当前 Git checkout 更新状态。
- **更新 checkout:** 仅当检查结果可 fast-forward 更新时展示更新入口，用户确认后执行更新并重新扫描。

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 扫描失败 | 页面错误摘要 | 重新扫描或返回 Home |
| 技能目录缺失 | 空状态 | 进入 Skills 创建 |
| 存在可归并技能 | 下一步任务 | 进入 Skills 归并 |
| 目标漂移检查失败 | 摘要警告 | 进入 Destinations 重试 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 初次进入 | 保留项目名称和路径，显示扫描中 |
| 重新扫描 | 禁用重复扫描按钮，保留旧摘要直到新结果返回 |
| 页面跳转 | 立即切换到目标页面并复用当前项目快照 |
