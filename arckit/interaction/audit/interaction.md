# Audit - 交互规范

## 交互策略

### 核心任务

用户在 Audit 页面查看技能审计发现，定位问题文件，并在修复后重新扫描验证。

### 主路径

页面展示审计评分、严重度过滤、发现列表和当前发现详情。用户选择发现后查看路径、规则、说明和修复位置，并可跳转 Skills 页面编辑对应文件。

### 决策点

用户决定按严重度、规则或技能筛选发现，决定跳转修复还是忽略非阻断项。系统区分阻断、警告和建议。

### 信息揭示

页面先展示评分和阻断摘要，再展示发现列表。详情区展示规则、证据、路径、行号、影响范围和修复入口。

### 状态流

扫描完成后显示发现列表。无发现时显示通过状态。重新扫描时保留旧结果并显示刷新状态。扫描失败时保留旧结果和错误原因。

### 反馈与恢复

用户从 Audit 跳转 Skills 后保留发现上下文。重新扫描成功后更新发现状态。扫描失败不清空旧发现。

### 输入输出边界

输入包括严重度过滤、规则过滤、发现选择、重新扫描和跳转修复。输出为审计评分、发现列表、规则详情和修复入口。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 发现列表 | 审计完成且存在发现 | FindingList |
| 详情 | 用户选择发现 | FindingDetail |
| 通过 | 无审计发现 | ContentUnavailableView |
| 重新扫描 | 用户点击 Rescan | ProgressView |
| 扫描失败 | 审计失败 | InlineError |

## 交互行为

- **过滤发现:** 用户选择严重度或规则后列表过滤。
- **选择发现:** 用户点击发现后详情区展示规则、证据和路径。
- **跳转修复:** 用户点击 Fix in Skills 后进入 Skills 并定位文件。
- **重新扫描:** 用户修复后点击 Rescan，系统刷新审计结果。
- **查看规则:** 用户点击规则名称后展示规则说明。

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 审计失败 | 页面顶部错误 | 重新扫描 |
| 文件已移动 | 详情错误 | 重新扫描 |
| 规则加载失败 | 详情占位 | 重试 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 重新扫描 | 保留旧发现并展示刷新中 |
| 详情加载 | 列表保持可用 |
| 跳转 Skills | 带上文件路径和行号上下文 |
