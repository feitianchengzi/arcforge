# 共享发布交互

## 策略基线

共享发布流程把高风险的远端写入拆成“先生成计划，再执行共享”。用户先输入远端仓库和目标路径模式，查看私有或公开发布计划，再触发 Git 同步和推送。

## 策略锚点

- 共享执行前必须展示远端地址、目标路径模式和提交信息。
- 没有远端地址或目标路径模式时，共享按钮不可执行。
- 按项目名写入模式必须要求项目文件夹名。
- 发布计划不修改远端仓库，执行共享才修改远端。
- 共享结果必须展示执行消息、提交状态、推送状态和目标路径。

## 状态机

### 表单输入状态

共享页左侧展示远端仓库输入、目标路径模式、可选项目文件夹名、提交信息和可见性操作。

远端仓库输入会延迟保存到工作区配置。项目文件夹名同样延迟保存。保存失败时，表单值保留在页面中。

用户选择直接路径时，说明当前仓库路径就是 Skill 项目根路径。用户选择按项目名写入时，页面展示项目文件夹名字段。

### 发布计划状态

用户点击私有团队仓库或公开发布后，系统生成发布计划。计划区域展示桌面端使用流程、CLI 安装命令和检查项。

没有生成计划时，计划区域显示空状态。切换项目或重新扫描后，已有计划清空。

公开发布计划强调移除内部路径和私有信息。私有发布计划强调权限和评审负责人。

### 共享执行状态

用户点击共享后，系统保存共享设置，扫描当前工作区，准备共享仓库检出目录，复制项目文件，写入 README 使用说明，提交并推送。

执行期间表单保持可见。执行完成后，页面展示共享输出，包括远端地址、分支、目标路径、是否提交、是否推送和 Git 消息。

远端没有文件变化时，共享结果展示没有文件变更，但仍展示推送结果。

### 共享失败状态

远端地址格式错误、项目名缺失、Git 克隆失败、提交失败或推送失败时，系统展示错误状态。用户输入的远端地址、目标模式、项目名和提交信息保留。

用户可以修正输入后重新共享，也可以先生成计划确认安装引用。

### 下载来源状态

添加项目弹窗中的 GitHub 来源下载是共享发布的入口前置流程。用户输入来源后，系统下载项目到本地缓存，成功后打开项目，失败后保留失败项目记录。

## 触发与反馈

私有团队仓库按钮设置可见性为私有并生成计划。

公开发布按钮设置可见性为公开并生成计划。

共享按钮在远端地址、目标模式和必要项目名齐备后可用。

目标模式切换立即保存共享设置，并更新目标模式说明。

远端地址和项目名输入使用短延迟保存，避免每次输入都触发配置写入。

## 容错与恢复

远端地址为空时不允许共享。远端地址格式错误时，错误反馈保留在当前页。

按项目名写入但项目名为空时不允许共享。

Git 不可用或认证失败时，共享失败反馈保留执行上下文，用户可以修复环境或凭据后重试。

生成计划失败不影响表单输入。用户可以重新扫描项目或修正配置后再次生成。

## 信息优先级

共享表单优先展示远端地址和目标路径模式，其次展示提交信息和可见性操作。

计划输出优先展示桌面端使用流程和安装命令，再展示检查项。

共享输出优先展示最终目标和执行结果，再展示详细 Git 消息。

## 平台语义

共享表单对应 `Form`，目标模式对应 `Picker(segmented)`，计划和执行日志对应 `ScrollView` 中的分组内容。下载来源弹窗对应 `Sheet`。
