# Sharing - 交互规范

## 交互策略

### 核心任务

用户为当前 Skill 项目生成发布计划，并在确认远端仓库、目标路径模式和提交信息后执行 GitHub-first 共享。

### 主路径

用户输入远端来源，选择直接路径或按项目名写入，生成私有或公开发布计划，确认安装命令和检查项后点击 Share now。系统同步共享工作树、写入 README 区块、提交并推送。

### 决策点

用户决定远端仓库、可见性、目标路径模式和项目文件夹名。系统通过计划输出、目标模式说明和按钮可用性帮助用户避免错误共享。

### 信息揭示

共享表单优先展示远端仓库和目标路径模式。计划输出优先展示桌面端使用流程和安装命令，再展示检查项。共享输出优先展示分支、目标路径、提交状态、推送状态和 Git 消息。

### 状态流

初始状态没有发布计划。用户生成计划后进入计划内容状态。共享执行时进入加载状态。共享成功后展示结果日志。远端地址无效、项目名缺失、Git 失败或推送失败时进入错误状态，并保留输入。

### 反馈与恢复

共享按钮在远端地址、目标模式和必要项目名齐备前不可用。共享失败保留远端地址、目标模式、项目名和提交信息。用户修正后可以重新生成计划或再次共享。

### 输入输出边界

输入包括远端仓库、可见性、提交信息、目标模式和项目名。输出包括发布计划、安装命令、检查项和共享结果。发布计划不写远端；共享执行写入远端。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 加载中 | 生成计划或执行共享 | ProgressView / Form |
| 成功（内容） | 发布计划或共享结果存在 | ScrollView / Form |
| 空状态 | 未生成发布计划 | ContentUnavailableView |
| 错误 | 远端解析失败、Git 失败或项目名缺失 | Alert / ContentUnavailableView |

## 导航流程

### 进入方式

用户从项目页签进入 Share。页面使用当前工作区快照和配置。

### 页面内导航

用户在共享表单、计划输出和共享输出之间纵向浏览。GitHub 来源下载从添加项目弹窗进入。

### 离开方式

用户切换到其他页签或切换项目。未完成共享的表单输入保留在当前页面状态。

## 交互行为

### 共享表单

- **输入远端仓库:** 系统延迟保存 teamRepo 到配置。
- **切换目标模式:** 系统保存 shareTargetMode 并更新说明。
- **输入项目名:** 系统延迟保存 shareProjectName。
- **点击 Private team repo:** 系统生成私有发布计划。
- **点击 Public release:** 系统生成公开发布计划。
- **点击 Share now:** 系统执行共享并展示日志。

### 计划输出

- **查看安装命令:** 用户复制或参考 skillshare 与 npx skills 命令。
- **查看检查项:** 用户确认审计、README、权限或公开发布清理要求。

### 来源下载

- **点击 Download:** 系统克隆或更新来源缓存，成功后打开项目根目录。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| GitHub source | 添加项目弹窗中选择 GitHub 来源 | Sheet | 来源输入、下载按钮、下载状态、失败提示 |
| Share error | 共享执行失败 | Alert | 错误消息和保留输入提示 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 远端地址为空 | Share now 禁用 | 输入远端仓库 |
| 项目名为空 | Share now 禁用 | 输入项目文件夹名或切换直接路径 |
| 来源下载失败 | 下载弹窗错误状态 | 修正地址或凭据后重试 |
| Git 命令失败 | 共享输出错误状态 | 修复 Git 环境或权限后重试 |
| 发布计划失败 | 计划区域错误状态 | 重新扫描或修正配置 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 生成计划 | 保留表单，计划区域显示加载 |
| 执行共享 | 保留表单和最近计划，共享输出显示处理中 |
| 下载来源 | 添加项目弹窗和侧边栏临时项目同时反馈 |
