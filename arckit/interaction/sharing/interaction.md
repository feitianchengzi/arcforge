# Sharing - 交互规范

## 交互策略

### 核心任务

用户为当前 Skill 项目生成发布计划，并在桌面端或 CLI 中确认远端仓库、GitHub 权限、系统推荐交付方式、目标路径模式和提交信息后执行 GitHub-first 共享。

### 主路径

桌面端用户输入远端来源，选择直接路径或按项目名写入，生成私有或公开发布计划。系统检测 GitHub 权限并自动选择推荐交付方式。用户确认安装命令、检查项、分支、提交、推荐交付方式和 Pull Request 信息后点击 Share now。系统同步共享工作树、写入 README 区块、提交、推送，并在交付方式要求时创建 Pull Request。

CLI 用户先运行计划或预检命令获得 JSON 结果。用户带确认参数执行共享命令后，系统使用同一权限规则、目标路径规则和交付方式执行提交、推送和 Pull Request 创建。

### 决策点

用户决定远端仓库、可见性、目标路径模式、项目文件夹名和配置组。系统根据 GitHub 权限自动选择目标仓库 Pull Request、fork Pull Request、直接推送共享分支或本地分支。系统通过计划输出、目标模式说明、权限说明、执行确认和按钮可用性帮助用户避免错误共享。CLI 可显式传入交付方式，用于自动化或恢复场景。

### 信息揭示

共享表单优先展示远端仓库、权限状态和目标路径模式。计划输出优先展示桌面端使用流程、CLI 命令和安装命令，再展示检查项。执行确认优先展示目标仓库、共享分支、交付方式、变更摘要和 Pull Request 标题。共享输出优先展示 Pull Request 链接、分支、目标路径、提交状态、推送状态和 Git 消息。

### 状态流

初始状态没有发布计划。用户生成计划或 CLI 预检后进入计划内容状态。权限检测完成后进入权限结果状态。共享执行前进入确认状态。共享执行时进入加载状态。共享成功后展示结果日志和 Pull Request 链接。远端地址无效、项目名缺失、GitHub 未登录、权限不足、Git 失败、推送失败或 Pull Request 创建失败时进入错误状态，并保留输入、本地分支和提交信息。

### 反馈与恢复

共享按钮在远端地址、目标模式和必要项目名齐备前不可用。远端写入动作需要用户确认。共享失败保留远端地址、目标模式、项目名、提交信息、系统选择的交付方式、共享分支和本地提交。用户修正后可以重新生成计划、重新检测权限、改用 CLI 指定 fork Pull Request、本地分支或再次共享。

### 输入输出边界

桌面端输入包括远端仓库、可见性、提交信息、目标模式、项目名、配置组和确认参数。CLI 输入可以额外包含共享分支和交付方式。输出包括发布计划、权限结果、系统选择的交付方式、安装命令、桌面端流程、CLI 命令、检查项、共享结果和 Pull Request 链接。发布计划不写远端；共享执行根据交付方式写入远端或只保留本地分支。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 加载中 | 生成计划、检测权限或执行共享 | ProgressView / Form |
| 成功（内容） | 发布计划、权限结果、CLI 预检或共享结果存在 | ScrollView / Form |
| 执行确认 | 用户点击 Share now 且存在可执行交付方式 | ConfirmationDialog / Form |
| 空状态 | 未生成发布计划 | ContentUnavailableView |
| 错误 | 远端解析失败、GitHub 未登录、权限不足、Git 失败、PR 失败或项目名缺失 | Alert / ContentUnavailableView |

## 导航流程

### 进入方式

用户从项目页签进入 Share。页面使用当前工作区快照和配置。

### 页面内导航

桌面端用户在共享表单、权限结果、计划输出、执行确认和共享输出之间纵向浏览。GitHub 来源下载从添加项目弹窗进入。CLI 用户在终端中通过计划、预检和执行命令推进同一状态流。

### 离开方式

用户切换到其他页签或切换项目。未完成共享的表单输入保留在当前页面状态。

## 交互行为

### 共享表单

- **输入远端仓库:** 系统延迟保存 teamRepo 到配置。
- **检测 GitHub 权限:** 系统展示登录状态、仓库权限、推荐交付方式和不可用原因。
- **切换目标模式:** 系统保存 shareTargetMode 并更新说明。
- **输入项目名:** 系统延迟保存 shareProjectName。
- **确定交付方式:** 系统根据 GitHub 权限自动选择目标仓库 PR、fork PR、直接推送共享分支或本地分支，并在执行确认中展示。
- **点击 Share now:** 系统生成私有共享计划并检测 GitHub 权限。
- **点击 Share now:** 系统展示执行确认。
- **确认执行:** 系统执行共享并展示日志、分支、提交和 Pull Request 链接。

### 计划输出

- **查看安装命令:** 用户复制或参考 skillshare 与 npx skills 命令。
- **查看 CLI 命令:** 用户复制或参考计划、权限预检和确认执行命令。
- **查看检查项:** 用户确认审计、README、权限或公开发布清理要求。

### CLI 流程

- **运行计划命令:** 系统输出发布计划、权限检查结果、推荐交付方式和执行预览 JSON。
- **运行确认执行命令:** 系统执行共享，输出共享结果 JSON。
- **权限不足:** 系统输出错误阶段、可用交付方式和手动恢复命令。

### 来源下载

- **点击 Download:** 系统克隆或更新来源缓存，成功后打开项目根目录。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| GitHub source | 添加项目弹窗中选择 GitHub 来源 | Sheet | 来源输入、下载按钮、下载状态、失败提示 |
| Share confirmation | 用户点击 Share now | ConfirmationDialog | 目标仓库、共享分支、系统选择的交付方式、变更摘要、PR 标题 |
| Share error | 共享执行失败 | Alert | 错误消息和保留输入提示 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 远端地址为空 | Share now 禁用 | 输入远端仓库 |
| 项目名为空 | Share now 禁用 | 输入项目文件夹名或切换直接路径 |
| GitHub 未登录 | 权限区域错误状态 | 登录 GitHub 或改用本地分支 |
| 无写权限但可 fork | 权限区域提示系统将使用 fork PR | 确认 fork PR 或改用 CLI 显式选择其它方式 |
| 无仓库权限 | 权限区域提示系统将使用本地分支 | 生成本地分支或修正仓库权限 |
| 来源下载失败 | 下载弹窗错误状态 | 修正地址或凭据后重试 |
| Git 命令失败 | 共享输出错误状态 | 修复 Git 环境或权限后重试 |
| Pull Request 创建失败 | 共享输出错误状态 | 使用已推送分支手动创建 PR 或重试 |
| 发布计划失败 | 计划区域错误状态 | 重新扫描或修正配置 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 生成计划 | 保留表单，计划区域显示加载 |
| 检测权限 | 保留表单，权限区域显示加载 |
| 执行共享 | 保留表单、最近计划和执行确认，共享输出显示处理中 |
| 下载来源 | 添加项目弹窗和侧边栏临时项目同时反馈 |
