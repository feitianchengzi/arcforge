# 应用外壳交互

## 策略基线

应用外壳以“先确定 Skill 项目，再执行项目内任务”为节奏。左侧项目列表固定承载项目上下文，顶部状态区承载全局操作反馈，页签承载项目内任务切换。

## 策略锚点

- 当前项目在页签切换时保持不变。
- 扫描、初始化、下载和环境检测的反馈出现在顶部状态区或项目列表中。
- 添加项目失败后保留来源地址或失败项目记录。
- Git 不可用时，下载和共享相关风险必须在顶部可见。

## 状态机

### 未选择项目

应用启动后没有可用当前项目时，主内容区展示添加项目空状态。用户可以打开添加项目弹窗，并选择本地目录或输入 GitHub 来源。

### 本地项目选择中

用户点击本地项目入口后，系统打开目录选择器。用户取消选择时，顶部状态显示取消信息，并保留未选择项目状态。

### GitHub 来源下载中

用户输入来源地址并点击下载后，侧边栏创建临时项目记录。临时记录显示下载中状态，当前主内容区显示下载说明和来源地址。

下载完成后，系统打开下载得到的项目根目录并执行扫描。下载失败时，临时记录变为失败状态，展示错误信息和可重新添加项目入口。

### 项目扫描中

打开项目或点击重新扫描后，顶部状态显示扫描中。扫描完成后，系统更新项目快照、最近项目记录、审计评分和页签内容。

扫描失败时，顶部状态显示错误，当前项目路径保留，用户可以重新扫描或切换项目。

### 环境检测

应用启动后检测 Git 环境。Git 可用时，顶部显示运行环境可用及版本。Git 不可用时，顶部显示 Git 缺失提示，下载和共享相关操作仍可进入但失败由操作自身反馈。

### 设置

用户点击设置后打开设置弹窗。语言切换立即作用于界面文案，并持久化到本地状态。关闭弹窗后返回当前项目上下文。

## 触发与反馈

添加 Skill 项目按钮打开添加项目弹窗。弹窗关闭不改变当前项目。

重新扫描按钮触发工作区扫描。按钮仅在存在当前项目且项目不处于下载失败或下载中状态时可执行。

初始化配置按钮触发默认配置写入和重新扫描。按钮仅在存在当前项目且项目不处于下载失败或下载中状态时可执行。

侧边栏项目项点击后打开对应项目。下载中或失败项目点击后只切换到临时状态，不触发扫描。

移除项目按钮删除最近项目记录。移除当前项目时，主内容区回到未选择状态。

## 容错与恢复

下载失败保留失败记录和错误信息，用户可以删除记录后重新添加。

扫描失败保留当前项目路径，用户可以重新扫描、初始化配置或选择其他项目。

Git 环境检测失败不阻塞本地扫描、审计和配置组操作。

## 信息优先级

项目列表优先展示项目名称，其次展示技能数量和审计评分。下载状态和错误状态替代健康摘要。

顶部状态优先展示当前项目名、操作状态和环境提示，其次展示重新扫描和初始化操作。

未选择状态只展示添加项目所需的最小目录结构，不展示项目内页签。

## 平台语义

应用外壳使用 macOS 桌面工作台语义。根布局对应 `NavigationSplitView`，侧边栏对应 `List(sidebar)`，顶部状态与操作区对应 `toolbar`，添加项目与设置对应 `Sheet`。
