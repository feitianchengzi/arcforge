# Profiles - 交互规范

## 交互策略

### 核心任务

用户在 Profiles 页面创建、编辑和审阅项目配置组，定义哪些技能作为一组被应用或共享。

### 主路径

页面展示配置组列表、配置组详情和技能选择范围。用户选择配置组后编辑名称、说明、技能集合和规则。保存后系统更新配置组并提示相关应用目标可能需要重新应用。

### 决策点

用户决定创建新配置组、修改已有配置组、删除配置组或调整技能范围。系统在删除和影响应用目标时要求确认。

### 信息揭示

列表先展示配置组名称、技能数量和被应用目标数量。详情区展示描述、技能选择、匹配规则和保存状态。

### 状态流

无配置组时显示空状态。选择配置组后进入编辑状态。修改后进入草稿状态。保存成功后刷新目标影响摘要。删除时进入确认。

### 反馈与恢复

未保存修改在离开页面或切换配置组前触发确认。保存失败保留草稿。删除失败保留配置组。

### 输入输出边界

输入包括配置组名称、描述、技能勾选、匹配规则、保存和删除。输出为配置组记录、影响目标提示和保存结果。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 列表 | 进入 Profiles 页面 | ProfileList |
| 编辑 | 用户选择配置组 | Form / SkillChecklist |
| 草稿 | 用户修改配置组 | DirtyIndicator |
| 删除确认 | 用户点击 Delete | ConfirmationDialog |
| 空状态 | 无配置组 | ContentUnavailableView |

## 交互行为

- **创建配置组:** 用户点击 New profile 后系统创建草稿。
- **选择技能:** 用户勾选技能后配置组进入未保存状态。
- **保存配置组:** 用户保存后系统写入配置组记录并更新影响摘要。
- **删除配置组:** 删除前展示关联应用目标数量。
- **查看影响:** 用户点击影响摘要后跳转 Destinations。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| Unsaved profile | 切换或离开 | ConfirmationDialog | 保存、放弃、取消 |
| Delete profile | 点击 Delete | ConfirmationDialog | 关联目标和删除影响 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 名称重复 | 表单错误 | 修改名称 |
| 保存失败 | 表单顶部错误 | 重试 |
| 删除失败 | 对话框错误 | 重试或取消 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 加载列表 | 保留页面骨架 |
| 保存 | 禁用重复提交 |
| 影响刷新 | 保存后局部刷新摘要 |
