# Destinations - 交互规范

## 交互策略

### 核心任务

用户在 Destinations 页面管理配置组应用目标和由 Skills 归并生成的应用关系，并检查来源与目标之间的漂移。

### 主路径

页面展示由 Skills 归并生成的应用关系，以及用户手动创建的应用目标组合。用户选择关系或目标后查看来源、配置组、目标路径、已应用技能和差异。用户可基于应用关系重新应用或检查漂移，也可执行普通配置组应用。

### 决策点

用户决定查看归并生成的应用关系、添加 Agent 目标或自定义项目目标、选择应用哪个配置组、是否在漂移后重新应用。系统在写入前展示计划。

### 信息揭示

列表展示应用目标组合和应用关系。详情区展示来源、配置组、目标路径、最近应用时间和差异摘要。高风险写入进入确认。

### 状态流

无应用关系和目标时显示空状态。添加目标进入 Sheet。应用前展示计划。应用中显示进度。漂移时展示差异，用户可查看详情或重新应用。

### 反馈与恢复

写入失败保留目标和计划。漂移检查失败保留上次已知状态。删除应用关系或目标只删除 SkillOps 记录，不删除目标文件。

### 输入输出边界

输入包括目标类型、目标路径、配置组、应用确认、漂移检查和删除。输出为应用关系记录、目标组合记录、应用结果、漂移报告和差异摘要。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 应用关系列表 | 进入 Destinations 页面 | AppliedRelationList |
| 添加目标 | 用户点击 Add destination | Sheet |
| 应用计划 | 用户选择配置组和目标 | PlanPreview |
| 漂移详情 | 目标与源不一致 | DiffView |
| 写入失败 | 应用失败 | InlineError |

## 交互行为

- **查看应用关系:** 用户选择由 Skills 归并生成的关系后，系统展示来源 Skill 项目、配置组和目标目录。
- **添加目标:** 用户选择 Agent 或自定义路径后系统验证可写性。
- **应用配置组:** 用户选择来源 Skill 项目配置组后系统展示写入计划。
- **保存关系:** 用户在一次性应用中选择保存后，系统创建应用关系。
- **重新应用:** 用户确认后系统把来源 Skill 项目配置组技能写入目标。
- **漂移检查:** 用户触发检查后系统对比来源 Skill 项目技能和目标文件。
- **删除记录:** 用户确认后系统只移除 SkillOps 应用关系或目标记录。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| Add destination | 点击 Add destination | Sheet | 目标类型、路径和配置组 |
| Apply plan | 点击 Apply | ConfirmationDialog | 写入文件列表和目标变化说明 |
| Drift diff | 点击 View diff | Sheet | 源文件与目标文件差异 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 目标不可写 | Sheet 错误 | 更换路径 |
| 应用失败 | 结果错误 | 重试 |
| 漂移检查失败 | 目标行错误 | 重新检查 |
| 来源项目缺失 | 详情错误 | 返回 Skills 修复归并关系 |
| 配置组缺失 | 详情错误 | 返回 Skills 或 Profiles 修复 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 应用中 | 显示文件级进度 |
| 漂移检查 | 保留旧状态并显示检查中 |
| 差异加载 | 先展示文件列表，再加载内容 |
