# Home - 交互规范

## 交互策略

### 核心任务

用户在 Home 页面选择最近项目、添加本地项目或下载远程 Skill 项目，并进入具体项目页面。

### 主路径

用户打开应用后看到最近项目列表和全局环境状态。用户选择已有项目时系统扫描并进入 Overview。用户点击 Add project 时选择本地目录或输入远程 Git/GitHub 地址，成功后进入扫描路径。

### 决策点

用户在本地目录和远程地址之间选择添加方式。系统通过项目名称、路径、技能数量、审计评分、下载状态和错误状态帮助用户判断项目是否可打开。

### 信息揭示

页面先展示最近项目和当前环境提示，再展示添加项目入口。添加项目 Sheet 展示本地目录入口、远程地址输入和失败恢复信息。

### 状态流

无项目时显示空状态。选择项目后进入扫描中状态。扫描成功后跳转到 Overview。下载远程来源时显示下载中状态。下载或扫描失败时保留项目记录和错误原因。

### 反馈与恢复

扫描失败保留路径和错误，用户可以重新扫描或移除记录。下载失败保留远程地址，用户可以修改后重试。移除项目只删除最近记录，不删除本地文件。

### 输入输出边界

输入包括最近项目点击、本地目录选择、远程地址、项目移除和重新扫描。输出为项目打开请求、下载结果、扫描状态、错误状态和环境提示。

## 页面状态

| 状态 | 触发条件 | data-kit 关键控件 |
|------|---------|-----------------|
| 空状态 | 无当前项目或最近项目为空 | ContentUnavailableView |
| 最近项目 | 存在最近项目记录 | Sidebar / List |
| 添加项目 | 用户点击 Add project | Sheet / DirectoryPicker |
| 下载中 | 远程来源下载中 | ProgressView |
| 错误 | 下载或扫描失败 | InlineError / Alert |

## 交互行为

- **打开项目:** 用户点击最近项目后，系统扫描目录并进入 Overview。
- **添加本地项目:** 用户选择目录后，系统扫描该目录。
- **添加远程项目:** 用户输入远程地址后，系统下载到缓存并打开本地项目根目录。
- **移除记录:** 用户确认后，系统删除最近项目记录。
- **修复 CLI:** 用户点击环境提示中的修复入口后，系统安装或更新 CLI shim。

## 弹窗/对话框

| 名称 | 触发 | data-kit | 内容 |
|------|------|----------|------|
| Add project | 点击 Add project | Sheet | 本地目录、远程地址、名称和错误 |
| Remove project | 点击移除 | ConfirmationDialog | 说明只移除记录 |
| Settings | 点击 Settings | Sheet | 语言与环境设置 |

## 错误处理

| 错误类型 | 展示方式 | 恢复操作 |
|---------|---------|---------|
| 目录取消 | 顶部状态 | 重新添加 |
| 下载失败 | Sheet 错误 | 修正地址后重试 |
| 扫描失败 | 项目行错误 | 重新扫描或移除 |
| Git 不可用 | 环境提示 | 安装 Git 后重试 |

## 加载策略

| 场景 | 策略 |
|------|------|
| 打开项目 | 保留 Home，显示扫描状态 |
| 下载远程来源 | 保留 Sheet 输入，显示下载中 |
| 环境检测 | 不阻塞项目选择 |
