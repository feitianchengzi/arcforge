# ArcForge - 品牌方向与视觉策略

## 元信息

- 状态: ✅ 已完成
- 视觉源样本: docs/assets/arcforge-overview.svg
- 适用范围: ArcForge desktop app、README product preview、后续视觉实现

## 品牌调性

ArcForge 的视觉气质是克制、可信、工程化和高效率。界面呈现为轻量运营控制台，而不是营销页或装饰性工具；用户应快速识别项目健康、风险状态、下一步操作和发布准备程度。

整体风格以浅色工作区为主，配合深色侧栏形成稳定的信息框架。蓝色只承担明确操作和选中状态，不大面积铺满页面；语义色用于表达审计结果和状态风险。

## 视觉策略

### 品牌气质

ArcForge 传达稳定、专业、面向开发工作流的工具感。视觉表达避免夸张渐变、插画、玻璃拟态和娱乐化装饰，保持接近 GitHub、Linear、Vercel Dashboard 一类工程工具的冷静语气。

### 信息层级

页面以“项目上下文、健康指标、下一步动作、风险发现”为主要层级。一级信息使用 26-28px 标题和清晰数字，二级信息使用 17-18px 面板标题，辅助路径、说明、标签和状态说明使用 12-14px 灰色文本。数字指标允许高权重和更大字号，用于快速扫描。

### 色彩角色

主操作色为清晰的工程蓝，用于主按钮、选中态、链接和关键行动。侧栏使用近黑蓝灰，承载项目列表和全局入口。内容区使用冷浅灰背景和白色面板，边框使用低对比灰。青色仅作为 logo 或品牌识别点。成功、警告、危险分别使用绿色、琥珀色和红色，优先用于审计和发布风险。

### 字体节奏

字体使用 Inter 优先，回退到系统 sans-serif。标题字重为 700-750，正文为 400-500，按钮和标签为 600-700。字号不随视口缩放；密集工具界面中避免过大的 hero 字体。代码、路径和 token 值使用等宽字体。

### 空间密度

空间策略为中等偏紧凑。页面外边距约 28-40px，面板内边距 16-24px，列表项高度 44-58px，按钮高度 38-44px。信息密度服务于工作台扫描，不使用大面积留白或卡片套卡片。

### 组件性格

组件边界清晰但轻量，圆角以 8px 为主，预览图中的 app shell 外框可使用 18-24px 作为展示容器。面板、指标卡、列表行使用 1px 边框和极轻阴影。按钮使用熟悉的矩形工具按钮样式，主按钮以蓝底白字表达确定操作，次按钮以白底灰边表达普通操作。

### 状态表达

选中态使用浅蓝背景、蓝色边框和深蓝文字。警告态使用浅琥珀背景和琥珀边框，危险态使用红色文字或浅红背景，成功态使用绿色文字或浅绿背景。状态表达应能在无图标时成立，图标作为增强而不是唯一信号。

### 主题策略

当前主主题为 light。深色主要用于侧栏和未来 dark theme，不改变信息层级和组件性格。主题变化只覆盖 token 值、对比关系和必要状态色，不改变密度、圆角、组件类型或页面结构。

## 色彩倾向

- 主色: 工程蓝，明确、直接、低装饰性。
- 品牌点色: 亮青色，仅用于 logo 或少量识别场景。
- 背景: 冷浅灰承载页面，白色承载内容面板。
- 侧栏: 深蓝黑，提供稳定导航锚点。
- 状态: 绿色、琥珀色、红色用于审计和风险，不扩展成大面积主题色。

## 字体倾向

- UI 字体优先使用 Inter，回退到系统 sans-serif。
- 代码、路径、命令和 token 值使用等宽字体。
- 标题与数字指标使用高字重，正文和说明保持普通字重。

## 设计原则

1. 工作流优先: 视觉层级服务于扫描、判断和行动。
2. 本地工具感: 界面像可长期打开的工作台，而不是短期营销页面。
3. 风险可见: 审计风险、发布状态和漂移结果必须通过颜色、文字和位置被快速识别。
4. 边界克制: 面板和列表用轻边框建立结构，不依赖重阴影或装饰背景。
5. 可投影: README 预览、桌面 UI 和后续主题都从同一组 tokens 与组件规格派生。

## 约束

- 主要平台是 Electron desktop app，同时服务 README product preview。
- 视觉实现应优先复用 tokens，不在业务组件中散落新色值。
- 卡片圆角默认不超过 8px，展示级外框可使用更大圆角。
- 不使用大面积紫色/蓝紫渐变、装饰圆球、玻璃拟态或强品牌插画。
