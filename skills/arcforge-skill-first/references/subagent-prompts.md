# 子代理 Prompt 模板

前测 skill 时使用这些模板。prompt 只提供任务本地上下文，不泄露预期答案。

先检查当前运行环境是否有可用的子代理工具或工具发现能力。Skill First 执行默认必须启动子代理，不需要用户额外确认；如果有可用工具，使用可用工具启动子代理，不在 skill 中硬编码具体命令。只有没有可用子代理工具，或工具调用失败且无法恢复时，才进入降级模拟：保存本应发送的 prompt，标注“未真实启动子代理”，再以独立使用者视角执行一次受限模拟。

## 首轮模拟

```text
你是一个独立的 Codex skill 使用者。

请使用这个 skill：<目标 skill 的绝对路径>

任务：
<真实用户请求>

工作区：
<绝对工作区路径>

你可以使用的临时路径：
<绝对临时路径列表>

规则：
- 除非任务明确要求修改源码，否则不要修改源文件。
- 不要 revert 其他人已有或并发的修改。
- 可以运行安全的本地命令。
- 模拟项目、fixtures、输出优先放在给定临时路径下。
- 先判断任务需要一个 skill 还是多个 skill，并说明本轮目标 skill 是否清楚。
- 先判断目标 skill 是否把自身表达成软件能力单元，而不只是 Markdown 文档。
- 不要只评估目标 skill 是否符合 `skill-creator` 的文件夹和资源结构；要用真实任务判断它是否能作为能力入口工作。
- 评估该任务是否需要 CLI、server、UI、状态、schema、测试 fixture 或 UI 回传机制；不需要时说明为什么。
- 评估目标 skill 的实现承载是否可能已经存在，例如仓库、服务、CLI、UI、MCP、脚本、schema 或 fixture；如果上下文不足，指出应该向用户询问什么。
- 不要把 CLI、server、UI、MCP、脚本或服务当作绕过 skill 的并行入口；它们只能作为某个 skill 的实现承载或依赖来评估。
- 如果需要写入操作，先说明原因，并保持范围受控。
- 创建或更新 skill 时，默认使用用户当前请求语言；除非用户明确要求其他语言。
- 如果 skill 验证通过，只评估它是否清楚引导到 ArcForge 治理下一步；不要执行真实 apply、share、push、目标目录覆盖或远程写入。

报告：
1. 你按 skill 走了哪些步骤。
2. 第一个不清楚或失败的点。
3. 哪些命令、路径或流程难以推断。
4. 任务需要的 skill 入口集合和本轮目标 skill 是否清楚。
5. 目标 skill 的软件能力单元建模是否充分。
6. 是否退回到只检查文件结构或资源清单，而没有验证真实任务体验。
7. 是否存在可能已有但未被发现或未被接入的实现承载。
8. 本轮应该接入已有实现、维护已有实现、包装进 skill，还是新建最小实现。
9. skill、CLI、server、UI、状态、schema、测试 fixture、回传机制或文档应该补充什么。
10. 如果需要 UI，UI 结果如何回到 agent 流程是否清楚。
11. 验证通过后，是否能清楚知道下一步该交给 ArcForge 的哪个阶段。
12. 你创建的临时路径。
```

## 复测模拟

```text
请使用更新后的 skill：<目标 skill 的绝对路径>，用新的临时路径重复同一个任务。

重点验证：
- 上一轮阻塞问题是否解决。
- 工作流是否仍然依赖隐藏上下文。
- 是否先识别任务需要哪些 skill 入口，再定位本轮目标 skill。
- 是否能先理解目标 skill 的软件能力单元形态，再执行具体维护。
- 是否会在目标 skill 的实现层发现已有承载，并在信息不足时向用户询问路径、仓库、服务或入口。
- 是否能区分接入已有实现、维护已有实现、包装进 skill 和新建最小实现。
- CLI、server、UI、状态、schema、测试或回传机制的选择是否清楚，且没有过度工程化。
- 命令和路径是否已经清楚。
- 验证通过后的 ArcForge 交接是否清楚，且没有绕过确认规则。
- 是否出现新的问题。

不要修改源文件。只报告剩余问题，以及目标工作流是否跑通。
```

## 降级模拟记录

无法启动子代理时，在临时目录保存：

```text
simulation-mode: degraded
reason: <为什么不能启动子代理>
availability-check: <检查了哪些工具或能力>
would-send-prompt: <原始子代理 prompt 路径或正文>
simulated-by: current-agent
limitations:
- 没有真正的独立上下文
- 结果只能发现明显摩擦点，不能替代真实子代理前测
```

降级模拟仍然要产出问题报告，并触发一次治理和复测；最终汇报必须说明这是降级模拟。

## 编码类 Skill 模拟

```text
请使用这个 skill：<目标 skill 的绝对路径>

任务：
<真实编码任务>

写入范围：
你只能编辑这些文件或目录：
<写入范围>

规则：
- 你不是独自在代码库中工作；不要 revert 无关修改。
- 修改保持在指定写入范围内。
- 先判断这个编码类 skill 是否需要 CLI、测试 fixture、状态文件、server 或 UI handoff 来支撑可复现执行。
- 先检查这些实现承载是否已经存在于当前项目或用户提供的其他入口，再决定是否新增。
- 有相关测试时运行测试。
- 创建或更新 skill 时，默认使用用户当前请求语言；除非用户明确要求其他语言。

报告修改过的文件、校验结果，以及 skill 指令中造成摩擦的地方。
```

## 只读类 Skill 模拟

```text
请使用这个 skill：<目标 skill 的绝对路径>

任务：
<真实只读任务>

规则：
- 不要编辑文件。
- 可以运行本地命令来检查上下文。
- 报告 skill 让什么变简单了、什么有歧义、什么无法完成，是否缺少 CLI、server、UI、状态、schema、测试或回传机制，以及目标 skill 是否可能已有实现承载但没有引导你去发现。
```
