# 子代理 Prompt 模板

前测 skill 时使用这些模板。prompt 只提供任务本地上下文，不泄露预期答案、诊断框架或修复思路。

子代理是普通执行者，不是评审者。它只负责按给定 skill 和边界执行真实任务，并把执行痕迹、选择、命令、路径、错误和卡点记录下来。观察是否有问题、判断问题类型、决定是否优化 skill、设计修复方案，都是主 agent 的职责。

先检查当前运行环境是否有可用的子代理工具或工具发现能力。Skill First 执行默认必须启动子代理，不需要用户额外确认；如果有可用工具，使用可用工具启动子代理，不在 skill 中硬编码具体命令。只有没有可用子代理工具，或工具调用失败且无法恢复时，才进入降级模拟：保存本应发送的 prompt，标注“未真实启动子代理”，再以独立使用者视角执行一次受限模拟。

## 首轮模拟

```text
你是一个独立的 Codex skill 使用者。你的职责是执行任务并记录过程，不负责评价 skill 是否需要优化。

请使用这个目标 skill：<目标 skill 的绝对路径>

任务：
<真实用户请求>

工作区：
<绝对工作区路径>

你可以使用的临时路径：
<绝对临时路径列表>

规则：
- 除非任务明确要求修改源码，否则不要修改源文件。
- 不要 revert 其他人已有或并发的修改。
- 可以运行安全的本地命令。
- 模拟项目、fixtures、输出优先放在给定临时路径下。
- 按给定目标 skill 执行任务；如果你发现自己想改用泛化 skill 或其他 skill，只记录你的选择冲突和不确定点，不要做架构分析。
- 不要评价目标 skill 是否符合 `skill-creator` 结构，也不要判断它是否需要优化。
- 不要分析 CLI、server、UI、MCP、脚本或服务应该如何设计；只记录你实际使用或想使用但找不到的入口。
- 如果需要写入操作，先说明原因，并保持范围受控。
- 如果任务涉及真实 apply、share、push、目标目录覆盖、远程写入或用户级目录写入，先停在计划、dry run、drift、预览或确认请求处，除非任务和边界明确允许写入。
- 不要提出 skill 修复建议；只报告执行过程和卡点。

报告：
1. 你最终选择或加载了哪些 skill。
2. 你按顺序执行了哪些步骤。
3. 你运行过或准备运行但停下来的命令。
4. 你读取、创建或写入过的路径。
5. 第一个不清楚、失败、需要确认或需要权限的点。
6. 你看到的原始错误、警告或不确定信息。
7. 如果你停在计划、dry run、drift 或确认请求处，说明下一步原本会写哪里。
8. 你创建的临时路径。
9. 你是否修改了文件；如果修改了，列出路径。
```

## 复测模拟

```text
请使用更新后的 skill：<目标 skill 的绝对路径>，用新的临时路径重复同一个任务。

重点验证：
- 上一轮阻塞问题是否解决。
- 你是否能按普通使用者视角完成或推进任务。
- 命令、路径、确认点是否比上一轮更清楚。
- 是否仍然在同一位置卡住。
- 是否出现新的错误、警告或权限请求。

不要修改源文件。只报告执行步骤、命令、路径、卡点和是否跑通；不要评价 skill 是否需要优化。
```

## 降级模拟记录

无法启动子代理时，在临时目录保存：

```text
simulation-mode: degraded
reason: <为什么不能启动子代理>
availability-check: <检查了哪些工具或能力>
would-send-prompt: <原始子代理 prompt 路径或正文>
simulated-by: current-agent
limitations:
- 没有真正的独立上下文
- 结果只能发现明显摩擦点，不能替代真实子代理前测
```

降级模拟仍然要产出执行记录，并由主 agent 做问题分类、治理和复测；最终汇报必须说明这是降级模拟。

## 编码类 Skill 模拟

```text
请使用这个 skill：<目标 skill 的绝对路径>

任务：
<真实编码任务>

写入范围：
你只能编辑这些文件或目录：
<写入范围>

规则：
- 你不是独自在代码库中工作；不要 revert 无关修改。
- 修改保持在指定写入范围内。
- 有相关测试时运行测试。
- 创建或更新 skill 时，默认使用用户当前请求语言；除非用户明确要求其他语言。
- 不要评价这个 skill 是否需要优化；只记录执行过程、修改和卡点。

报告修改过的文件、运行过的命令、校验结果、失败信息和卡住的具体位置。
```

## 只读类 Skill 模拟

```text
请使用这个 skill：<目标 skill 的绝对路径>

任务：
<真实只读任务>

规则：
- 不要编辑文件。
- 可以运行本地命令来检查上下文。
- 不要评价 skill 是否需要优化；只报告你尝试了什么、读取了什么、哪里不清楚、哪里无法继续、原始错误是什么。
```
