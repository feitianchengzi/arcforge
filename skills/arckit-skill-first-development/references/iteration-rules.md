# 迭代规则

收到子代理报告后使用本文件。

## 问题分类

把每个发现归类：

- Skill 表述问题：skill 缺少规则、顺序、例子、路径约定或停止条件。
- 语言不匹配：目标 skill 没有使用用户当前请求语言，或无理由默认使用英文。
- Agents 元数据不足：`agents/openai.yaml` 没有同步目标 skill 的关键输出格式、边界规则或语言要求。
- 正式 skill 处理问题：没有根据当前 agent 检查用户级和项目级 skill 目录，或直接修改了正式 skill 原始路径。
- 子代理启动问题：把用户额外确认作为启动子代理的前置条件，或有可用子代理工具时仍走降级模拟。
- 缺少 reference：细节需要复用，但不适合塞进 `SKILL.md`。
- 缺少工具能力：skill 需要 CLI、脚本、Desktop 或 API 支持，但当前不存在。
- 不安全流程：skill 允许破坏性、远程、凭证相关或状态性操作，但确认不足。
- 测试环境问题：模拟失败来自沙箱、依赖缺失、认证缺失或临时路径设置。
- 产品决策问题：需要用户决定策略、范围或产品方向。

## 修复策略

- 如果问题是歧义，优先修 skill 表述。
- 如果目标 skill 语言和用户请求语言不一致，优先统一 `SKILL.md`、references、`agents/openai.yaml` 的主要文案语言。
- 如果 `agents/openai.yaml` 只写了泛泛默认 prompt，补充关键输出格式、边界规则和语言要求。
- 如果发现目标来自正式 skill，先补齐“原始路径只读、复制到当前项目 `skills/<skill-name>/` 后再改”的流程。
- 如果子代理能力可用但流程要求额外确认，删除这个门槛；只有没有可用工具或工具不可恢复失败时才允许降级模拟。
- 如果细节可复用但过长，新增或更新 reference。
- 如果新增或更新 reference，确认主 `SKILL.md` 中有清楚的读取条件，否则新 agent 可能无法发现它。
- 只有当现有指令无法让流程可靠时，才补小型工具或 CLI 能力。
- 明确记录产品缺口，不要用隐性 workaround 掩盖。
- 修复保持范围小，不重写无关部分。

## 复测条件

以下情况需要再次启动子代理模拟：

- 首轮在核心流程完成前失败。
- 修改了命令顺序、路径规则、写入规则或停止条件。
- 修改了正式 skill 搜索路径、复制规则或写入边界。
- 修改了子代理启动条件或降级模拟条件。
- 修改了目标 skill 的语言规则或语言一致性要求。
- 修改了 `agents/openai.yaml` 的触发、默认 prompt 或关键行为约束。
- 新增了子代理必须能发现的 reference。
- 新增或修改了 skill 会使用的工具能力。
- 首轮只能做降级模拟，但你补充了真实子代理或降级模拟规则。

以下情况可以不复测：

- 只剩已接受的产品缺口。
- 用户要求停止。
- 校验需要凭证或外部系统，而用户没有批准。

## 质量标准

当满足以下条件时，可以认为该 skill 通过本轮前测：

- 新 agent 不依赖隐藏上下文也能识别正确工作流。
- 目标 skill 的主要内容语言与用户当前请求语言一致，除非用户明确指定其他语言。
- `agents/openai.yaml` 能帮助使用者启动正确任务，而不是只重复 skill 名称。
- Skill First 默认真实启动子代理；降级模拟只用于没有可用子代理能力或工具不可恢复失败。
- 第一个写入步骤有明确确认或明确写入范围。
- 每个 reference 都能从 `SKILL.md` 的读取条件中被发现。
- 常见路径约定明确。
- 正式 skill 的来源、复制位置和禁止直接修改原始路径的规则明确。
- 失败状态说明下一步怎么做。
- 最终输出说明发生了什么、改了什么、还不确定什么。
